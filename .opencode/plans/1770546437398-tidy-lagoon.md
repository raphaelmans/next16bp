# Fix Schedule & Pricing overlap below `xl`

## Goal
- Resolve overlapping/cramped Schedule & Pricing row fields (`Start`, `End`, `Open`, `Currency`, `Hourly rate`, etc.) on `md`/`lg` widths.
- Keep behavior stable with minimal code change and no data/model updates.

## Recommended implementation
- Update responsive breakpoint usage in `src/features/owner/components/court-schedule-editor.tsx` so card layout is used for all sizes below `xl`.
  - Change mobile card wrapper class from `md:hidden` to `xl:hidden`.
  - Change desktop table class from `hidden md:table` to `hidden xl:table`.
- Do not modify shared `Table` primitives (`src/components/ui/table.tsx`) to avoid app-wide side effects.

## Critical files
- Modify: `src/features/owner/components/court-schedule-editor.tsx`
- Verify (shared component usage):
  - `src/app/(owner)/owner/places/[placeId]/courts/[courtId]/schedule/page.tsx`
  - `src/app/(owner)/owner/places/[placeId]/courts/setup/page.tsx`

## Validation plan
1. Run `pnpm lint`.
2. Manual responsive checks on both route entry points at widths: `768`, `1024`, `1279`, `1280`, `1440`.
3. Confirm no overlap/cramping for row labels/inputs/actions.
4. Verify interactions still work in card mode and table mode:
   - Add block, edit start/end times, toggle open/closed
   - Change currency/rate, use `Copy to all`, remove block
   - Save action completes (`Save schedule` and `Save & Continue`)
5. Boundary check: confirm expected mode switch at `xl` (`1279` = cards, `1280` = table).

## Risk notes
- `md`/`lg` users will see card layout (more vertical scrolling) instead of dense table; this is intended to prevent overlap and preserve usability.
