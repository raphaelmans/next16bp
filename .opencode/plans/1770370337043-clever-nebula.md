# Chat Inbox Composition Refactor Plan

## Goal

Extract a shared inbox shell using composition patterns, then migrate reservation inbox and admin/support inbox onto it so we remove duplication and bring admin mobile UX to list-first/thread parity.

## Why this change

- `src/features/chat/components/chat-widget/reservation-inbox-widget.tsx` and `src/features/support-chat/components/support-inbox-widget.tsx` duplicate large UI sections (trigger, sheet, search/refresh, list/archive, thread panel).
- Reservation inbox already has mobile list-first/thread behavior, while support/admin inbox still stacks list+thread on mobile.
- We want composition-first architecture (explicit adapters) instead of introducing one giant `mode`/boolean-heavy component.

## Composition approach (vercel-composition-patterns)

- Use explicit feature adapters (reservation vs support) feeding a shared shell.
- Keep domain rules in adapters (archive logic, sorting, metadata hydration, labels).
- Keep shell domain-agnostic and focused on layout + interaction flow.

## Target Architecture

### Shared Shell

- `src/features/chat/components/inbox-shell/inbox-shell.tsx`
  - Renders floating trigger, sheet frame, list pane, thread pane.
  - Handles responsive composition behavior:
    - mobile: list OR thread pane,
    - desktop: split pane.
  - Accepts render callbacks/slots for list items and thread props.

- `src/features/chat/components/inbox-shell/inbox-shell.types.ts`
  - Defines shared contracts for sections/items, shell copy/config, and thread config.

### Reservation Adapter

- `src/features/chat/components/chat-widget/use-reservation-inbox-adapter.ts`
  - Keeps reservation-specific logic currently in widget:
    - stream query + sync state,
    - `getThreadMetas` hydration,
    - priority sorting,
    - archive/read-only rules,
    - reservation open event handling.
  - Preserves existing owner/player storage keys.

### Support/Admin Adapter

- `src/features/support-chat/components/use-support-inbox-adapter.ts`
  - Keeps support-specific logic:
    - `cr-`/`vr-` channel filtering,
    - support row labels,
    - current archive heuristic,
    - support storage keys.
  - Adds mobile pane behavior parity (list-first/thread with back).

### Thin Widget Entry Components

- `src/features/chat/components/chat-widget/reservation-inbox-widget.tsx`
  - Reduced to compose shell + reservation adapter.
- `src/features/support-chat/components/support-inbox-widget.tsx`
  - Reduced to compose shell + support adapter.

## Step-by-step implementation order

1) Extract shell UI from reservation widget into `inbox-shell.tsx` without changing visible behavior.
2) Introduce reservation adapter hook and wire reservation widget to shell + adapter.
3) Extract support adapter hook and wire support widget to the same shell.
4) Implement support mobile pane parity in adapter/view-model:
   - open to list,
   - row click switches to thread on mobile,
   - thread back button returns to list.
5) Remove dead duplicated code from both original widgets.
6) Keep `StreamChatThread` API as-is except passing existing `onBack` for support.
