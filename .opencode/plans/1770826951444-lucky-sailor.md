# Plan: Reduce Slow Spinner on Court Setup Schedule

## Goal
Fix intermittent long spinner time when opening:
`/owner/venues/[placeId]/courts/setup?courtId=...&step=schedule&from=setup`

User-confirmed repro path: this happens when navigating from `/owner/get-started`.

## Diagnosis
- In `src/app/(owner)/owner/places/[placeId]/courts/setup/page.tsx`, the page-level full-screen spinner blocks on `sportsLoading` even for `step=schedule`, where sports data is not needed.
- The same page also fetches hours/rules at the parent level while `CourtScheduleEditor` fetches them again for schedule UI; this increases startup query pressure.
- With `httpBatchLink` in `src/common/providers/index.tsx`, route transition bursts can feel intermittently slow because non-critical queries can delay visible content.

## Recommended Implementation

### 1) Make the setup page loading gate step-aware
**File:** `src/app/(owner)/owner/places/[placeId]/courts/setup/page.tsx`

- Split loading into:
  - **Route-critical blocking loading:** `orgLoading`, `placeLoading`, and `courtLoading` (when `courtId` exists).
  - **Step-local loading:** sports/hours/pricing data that should not block the whole page for schedule step.
- Keep the full-screen spinner only for route-critical loading.
- Remove `sportsLoading` from the full-screen blocking condition.

### 2) Fetch sports only when needed
**File:** `src/app/(owner)/owner/places/[placeId]/courts/setup/page.tsx`

- Change `trpc.sport.list.useQuery({})` to be conditionally enabled only when sports are required (details/create step).
- Keep behavior for details step unchanged (sports list still available when editing/creating details).

### 3) Stop parent-level schedule prefetch during schedule step
**Files:**
- `src/features/owner/hooks.ts`
- `src/app/(owner)/owner/places/[placeId]/courts/setup/page.tsx`

- Extend `useCourtHours` and `useCourtRateRules` to accept optional query options (at minimum `enabled`).
- In setup page, enable parent-level hours/rules queries only when `currentStep === "publish"` (where summary/prereq checks are needed).
- Keep `CourtScheduleEditor` as the data owner for schedule editing/loading.

### 4) Preserve schedule-step UX while data resolves
**File:** `src/features/owner/components/court-schedule-editor.tsx`

- Keep the editor-level spinner/fallback for hours/rules loading.
- Ensure the AppShell and surrounding page content appear as soon as route-critical data is ready.
