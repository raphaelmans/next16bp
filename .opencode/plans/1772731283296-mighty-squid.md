# Full System UI/UX Cutover & Revamp Plan

## Objective
Execute a hard cutover of the entire design system and UI layer to strictly use **Radix shadcn-ui components**, retaining only the **primary teal** color theme. This process will rigorously apply the full suite of diagnostic, quality, intensity, adaptation, enhancement, and system skills to achieve a world-class, cohesive, and resilient interface.

---

## Phase 1: Diagnostic & System Definition
**Skills applied:** `/teach-impeccable`, `/audit`, `/critique`, `/extract`

1. **System Setup (`/teach-impeccable`)**: Establish the persistent "Minimalist Bento" design guidelines with warm-tinted neutrals and primary teal.
2. **Global Audit & Critique (`/audit`, `/critique`)**: 
   - Identify all non-shadcn components in `src/components/ui/` (e.g., `button-group`, `spinner`, `field`, `empty`).
   - Identify "AI slop" or overly intense marketing sections (e.g., home page hero gradients).
   - Flag all hardcoded hex values across charts (`analytics/charts/`) and OpenGraph images.
3. **Extraction & Consolidation (`/extract`)**:
   - Consolidate all global CSS tokens in `src/app/globals.css`. Retain Primary Teal (`oklch(0.58 0.11 175)` / `~#0D9488`).
   - Map all non-standard tokens to shadcn semantic variables (background, foreground, muted, accent, destructive, card, popover).

---

## Phase 2: Standardization & Quality
**Skills applied:** `/normalize`, `/harden`, `/optimize`

1. **Strict Shadcn Normalization (`/normalize`)**:
   - Refactor custom or hybrid components to use pure shadcn-ui primitives. Use **shadcn MCP / Context7 MCP** to cleanly install/re-install components if standard versions are missing or outdated.
   - Refactor components with custom utility classes (like `Badge` success/warning variants) to use proper semantic CSS variables.
2. **Resilience & Error Handling (`/harden`)**:
   - Ensure all forms use the standard shadcn `Form` (React Hook Form + Zod).
   - Improve text-overflow handling in standard `Card` and `Table` components.
   - Standardize error states using shadcn `Alert` or `Toast` components.
3. **Performance (`/optimize`)**:
   - Ensure dynamic imports for heavy client components (like charts) and verify Radix UI bundle usage.

---

## Phase 3: Adaptation & Content
**Skills applied:** `/adapt`, `/clarify`, `/copywriting`, `/onboard`

1. **Responsive Parity (`/adapt`)**:
   - Ensure components adapt gracefully (e.g., using shadcn `Drawer` for mobile vs. `Dialog` for desktop where appropriate).
2. **Clarity & Copy (`/clarify`, `/copywriting`)**:
   - Rewrite auth flows and generic text. Transform passive states ("Create Account") into active, brand-aligned propositions ("Start Booking", "Set Up Venue").
   - Soften cooldown errors ("Please wait 60s before requesting a new code").
3. **First-Time User Experience (`/onboard`)**:
   - Redesign the Owner Onboarding wizard (`venue-step.tsx`, `complete-step.tsx`).
   - Replace dead-end empty states with actionable, illustrated `Card` or `EmptyState` patterns containing clear "Create" or "Add" CTAs.

---

## Phase 4: Intensity & Enhancement
**Skills applied:** `/distill`, `/quieter`, `/bolder`, `/colorize`, `/animate`, `/delight`, `/polish`

1. **Intensity Calibration (`/distill`, `/quieter`, `/bolder`)**:
   - **Distill & Quieter**: Strip out heavy, hardcoded `rgba` radial gradients and excessive box-shadows on the home page hero. Simplify the bento grid.
   - **Bolder**: Use the primary teal deliberately to highlight the primary booking/setup workflows, creating stark, confident contrasts against the warm neutral background.
2. **Visual & Motion Enhancement (`/colorize`, `/animate`, `/delight`)**:
   - **Colorize**: Re-color analytics charts to exclusively use the refined shadcn CSS variables.
   - **Animate & Delight**: Add purposeful micro-interactions using standard `tailwindcss-animate`. Ensure smooth transitions for dropdowns, dialogs, and button hovers to elevate the app from functional to delightful.
3. **Final Polish (`/polish`)**:
   - Execute a final alignment pass. Ensure padding, margins, flex gaps, and typography scales perfectly match the shadcn-ui baseline without rogue utility classes.

---

## Verification Strategy
- **Visual Smoke Test**: Run `pnpm dev` to manually verify the home page hero, auth pages, onboarding wizards, and charts.
- **MCP Validation**: Use shadcn MCP's `/shadcn_get_audit_checklist` tool if available, ensuring components map correctly.
- **Lint & E2E**: Run `pnpm lint` and `pnpm test:e2e` to guarantee no workflows or component boundaries were broken during the cutover.
