# Admin Tool: Revalidate Home Page (Featured Section)

## Goal
Add an admin-only page under `src/app/(admin)/admin/tools/` that lets an admin trigger on-demand revalidation of the public home page (`/`). This fixes cases where the homepage still shows stale `featuredPlaces` due to `export const revalidate = 3600` ISR caching.

## Background / Root Cause
- `src/app/page.tsx` uses ISR (`export const revalidate = 3600`) and server-side data (`publicCaller.place.list({ featuredOnly: true, ... })`).
- DB currently has no rows with `featured_rank > 0`, but the homepage can still render old featured data until the ISR cache is invalidated.
- There is currently no on-demand revalidation endpoint/util in the repo.

## Approach (Recommended)
Use a Next.js **Server Action** called from an admin Tools page.

Why this approach:
- `revalidatePath` is explicitly supported in Server Functions (Server Actions).
- Auth is straightforward: call `requireAdminSession()` inside the action.
- No need to add new public API routes or new tRPC routers.

## Files To Add
1. `src/app/(admin)/admin/tools/revalidate/page.tsx`
2. `src/app/(admin)/admin/tools/revalidate/actions.ts`

## Implementation Details

### 1) Server Action
File: `src/app/(admin)/admin/tools/revalidate/actions.ts`

- Add `"use server"`.
- Import:
  - `revalidatePath` from `next/cache`
  - `requireAdminSession` from `@/lib/shared/infra/auth/server-session`
  - `appRoutes` from `@/common/app-routes` (for `/` constant)
- Validate a confirm input (simple `zod` schema recommended) to avoid accidental clicks:
  - input: `{ confirm: true }`
- Security:
  - Call `await requireAdminSession("/admin/tools/revalidate")` (or similar) inside the action.
- Behavior:
  - `revalidatePath(appRoutes.index.base)`
  - Return a small payload e.g. `{ ok: true, path: "/", at: Date.now() }`.

### 2) Admin Tools UI Page
File: `src/app/(admin)/admin/tools/revalidate/page.tsx`

- Make it a client component (like `notification-test`) for toasts and UX.
- Reuse existing admin shell:
  - `AppShell`, `AdminSidebar`, `AdminNavbar`, `PageHeader`, `Card`, `Button`, `Checkbox`.
- Hooks:
  - `useSession()`, `useLogout()`, `useAdminSidebarStats()` to match other admin pages.
- UI elements:
  - Confirm checkbox: “I understand this will purge the cached homepage and affect what users see.”
  - Button: “Revalidate / (Home)”
  - On click: call the server action inside `startTransition` / `useTransition`.
  - Success toast: show returned payload.
  - Optional convenience: secondary button/link to open `/` in a new tab after revalidate.

## Notes / Expected Behavior
- Revalidation happens on the next visit to `/` (it marks the path for revalidation). To verify, open `/` after triggering the action (hard refresh / incognito).
