# Admin Tool: Tag-Based Revalidate (Home Featured)

## Goal
Add cache tags for the homepage featured list and expose an admin-only trigger that can revalidate via `revalidateTag` (so CDN cache tag purge is possible). Keep the existing path-based revalidate as a fallback.

## Background / Root Cause
- `src/app/page.tsx` uses ISR (`export const revalidate = 3600`) and server-side data (`publicCaller.place.list({ featuredOnly: true, ... })`).
- DB currently has no rows with `featured_rank > 0`, but the homepage can still render old featured data until the ISR cache is invalidated.
- There is currently no on-demand revalidation endpoint/util in the repo.

## Approach (Recommended)
Use a Next.js **Server Action** called from the admin Tools page, add a tagged cache around the featured list fetch, and trigger `revalidateTag` from the tool.

Why this approach:
- `revalidateTag` is supported in Server Functions.
- Tagging provides a stable string you can use with the CDN purge UI.
- No public API routes needed; admin-only server action remains.

## Files To Update
1. `src/app/page.tsx`
2. `src/app/(admin)/admin/tools/revalidate/actions.ts`
3. `src/app/(admin)/admin/tools/revalidate/page.tsx`

## Implementation Details

### 1) Tag the Featured List Cache
File: `src/app/page.tsx`

- Wrap the featured list fetch in `unstable_cache` with tag `home:featured`.
- Keep `export const revalidate = 3600` for ISR (tag will allow manual invalidation).
- Example approach:
  - `const getFeaturedPlaces = unstable_cache(async () => { ...publicCaller.place.list(...)... }, ["home-featured"], { tags: ["home:featured"] });`
  - Use `await getFeaturedPlaces()` in the page.

### 2) Server Action (Revalidate Tag + Path)
File: `src/app/(admin)/admin/tools/revalidate/actions.ts`

- Add `revalidateTag("home:featured")` alongside `revalidatePath("/")`.
- Return payload should include `tag` so the UI can display it.

### 3) Admin Tools UI Page
File: `src/app/(admin)/admin/tools/revalidate/page.tsx`

- Make it a client component (like `notification-test`) for toasts and UX.
- Reuse existing admin shell:
  - `AppShell`, `AdminSidebar`, `AdminNavbar`, `PageHeader`, `Card`, `Button`, `Checkbox`.
- Hooks:
  - `useSession()`, `useLogout()`, `useAdminSidebarStats()` to match other admin pages.
- UI elements:
- Confirm checkbox: “I understand this will purge the cached homepage and affect what users see.”
- Button: “Revalidate / (Home)”
- On click: call the server action inside `startTransition` / `useTransition`.
- Success toast: show returned payload.
- Display the cache tag string (e.g., `home:featured`) so admins can also purge by tag in the CDN UI.

## Notes / Expected Behavior
- `revalidateTag` marks the tagged data stale; the next request to `/` will fetch fresh featured data.
- `revalidatePath("/")` remains as a safety net for path ISR.
