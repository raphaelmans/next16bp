# Plan: Enhanced Wizard Navigation Visual Cues

## Problem Statement

The setup wizard lacks concrete/tangible visual cues for navigation on mobile (and to a lesser extent, desktop):
1. **Progress bar shows only numbered circles on mobile** - step labels are hidden (`hidden sm:inline`)
2. **No "Step X of Y" indicator** - users don't know their position in the flow
3. **Navigation buttons lack visual prominence** - ghost variant with small size
4. **Touch targets may be below 44x44px minimum** per UX guidelines

## UX Research Insights

| Guideline | Current State | Required Fix |
|------------|---------------|--------------|
| "Step 2 of 4 indicator" | Missing | Add explicit step count |
| Min 44x44px touch targets | Buttons are `size="sm"` | Increase button size on mobile |
| Min 8px gap between targets | `gap-2` (8px) | ✓ Already compliant |
| Visual feedback on hover/interaction | Ghost buttons lack prominence | Make Continue primary, improve Back visibility |

## Design Approach

### 1. WizardProgress Enhancement (Top Bar)

**Current:**
```
[1]──[2]──[3]──[4]──[5]──[6]   (mobile: numbers only, labels hidden)
```

**New:**
```
Step 2 of 6
    •  ○  ○  ○  ○  ○   ( dots with current highlighted )
    Venue                  ( current step label )
```

Changes:
- Add "Step X of 6" counter as compact text header (all viewports)
- Add current step label below progress dots (all viewports)
- Replace numbered circles with simpler dots (cleaner, less clutter)
- Keep completion checkmarks on passed steps
- Keep clickable behavior for navigation

### 2. WizardNavigation Enhancement (Bottom Bar)

**Current:**
- Back: `ghost` `sm` with ArrowLeft icon
- Skip: `ghost` `sm` with SkipForward icon
- Continue: `default` `sm` with ArrowRight icon

**New:**
- Back: Increase to `default` size on mobile, add `cursor-pointer`, more visible hover state
- Skip: Keep `ghost` but increase to `default` size
- Continue: Use `default` variant with full prominence, `min-h-[44px]` on mobile
- Add subtle affordance arrow hints (chevrons pointing direction)

### 3. Responsive Touch Target Compliance

Ensure all interactive elements meet 44x44px minimum on mobile:
- Progress dots: `h-11 w-11` with centered `h-3 w-3` visual dot
- Back/Skip/Continue buttons: `min-h-[44px]` on mobile

```tsx
// New structure:
<div className="sticky top-0 z-30 border-b bg-background/95 backdrop-blur">
  {/* Step counter header */}
  <div className="text-center text-xs text-muted-foreground pt-3">
    Step {currentStepIndex + 1} of {totalSteps}
  </div>
  
  {/* Progress dots */}
  <div className="flex items-center justify-center gap-2 px-4 py-2">
    {steps.map((step, i) => (
      <button className="h-11 w-11 flex items-center justify-center cursor-pointer">
        <div className={cn(
          "h-3 w-3 rounded-full transition-colors",
          isComplete && "bg-primary",
          isCurrent && "h-3.5 w-3.5 ring-2 ring-primary ring-offset-2",
          !isComplete && !isCurrent && "bg-muted-foreground/30"
        )}>
          {isComplete && <Check className="h-2.5 w-2.5" />}
        </div>
      </button>
    ))}
  </div>
  
  {/* Current step label */}
  <div className="text-center pb-3">
    <span className="text-sm font-medium">{currentStepConfig.label}</span>
  </div>
</div>
```

### File: `wizard-navigation.tsx`

```tsx
// Enhanced bottom bar:
<div className="sticky bottom-0 z-30 border-t bg-background/95 backdrop-blur">
  <div className="mx-auto flex max-w-2xl items-center justify-between gap-4 px-4 py-4">

          <ArrowRight className="ml-2 h-5 w-5" />
        </Button>
      )}
    </div>
  </div>
</div>
```
