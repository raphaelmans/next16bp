# OpenSpec Plan: PH-only locale simplification (temporary)

## Goal
- Temporarily hide locale-related fields from product flows:
  - `currency` -> default `PHP`
  - `country` -> default `PH` (Philippines)
  - `timeZone` -> default `Asia/Manila`
- Remove these fields from write APIs and owner/admin read APIs so clients cannot send or rely on them.
- Keep public/discovery/reservation read APIs unchanged in this pass to avoid broad regression.

## Confirmed Scope
- Included:
  - UI input removal (owner/admin)
  - Write contract removal (DTOs + client payloads)
  - Server-side default enforcement
  - Owner/admin read response redaction
- Excluded for now:
  - Public/discovery/reservation response contract redaction
  - DB schema migration (defaults already exist where needed)

## Implementation Plan

### 1) Canonical defaults (single source of truth)
- Add shared constants module:
  - `src/common/location-defaults.ts`
    - `DEFAULT_COUNTRY = "PH"`
    - `DEFAULT_TIME_ZONE = "Asia/Manila"`
    - `DEFAULT_CURRENCY = "PHP"`
- Replace scattered literals in touched files with these constants.

### 2) Remove fields from write contracts and enforce defaults server-side
- Update write DTOs (remove client-provided locale fields):
  - `src/lib/modules/place/dtos/place.dto.ts`
  - `src/lib/modules/court/dtos/create-curated-court.dto.ts`
  - `src/lib/modules/court/dtos/admin-update-court.dto.ts`
  - `src/lib/modules/court-rate-rule/dtos/court-rate-rule.dto.ts`
  - `src/lib/modules/court-addon/dtos/court-addon.dto.ts`
  - `src/lib/modules/place-addon/dtos/place-addon.dto.ts`
- Enforce defaults in service write paths:
  - `src/lib/modules/place/services/place-management.service.ts`
    - always persist `country=PH`, `timeZone=Asia/Manila`
  - `src/lib/modules/court/services/admin-court.service.ts`
    - remove update path mutability for `country/timeZone`
    - always persist defaults
  - `src/lib/modules/court-rate-rule/services/court-rate-rule.service.ts`
    - persist `currency=PHP` regardless of client payload
  - `src/lib/modules/court-addon/services/court-addon.service.ts`
    - persist addon currencies as `PHP` for hourly/flat pricing records
  - `src/lib/modules/place-addon/services/place-addon.service.ts`
    - persist addon currencies as `PHP` for hourly/flat pricing records

### 3) Redact owner/admin read responses
- Redact locale fields from management/admin reads (not public/discovery/reservation):
  - Owner management:
    - `placeManagement.list`
    - `placeManagement.getById`
    - `courtRateRule.get`
    - `courtAddon.get`
    - `placeAddon.get`
  - Admin management:
    - `admin.court.list`
    - `admin.court.getById`
    - `admin.claim.getById` (if place payload includes locale fields)
    - `admin.placeVerification.getById` (if place payload includes locale fields)
- Apply shaping/mapping in router/service return payloads so locale fields are omitted for these endpoints.

### 4) Remove locale controls from owner/admin UI and payload builders
- Owner forms/editors:
  - `src/features/owner/schemas.ts`
  - `src/features/owner/components/place-form.tsx`
  - `src/features/owner/components/place-form-hooks.ts`
  - `src/features/owner/components/place-form-helpers.ts`
  - `src/features/owner/hooks/places.ts`
  - `src/features/owner/components/court-pricing-editor.tsx`
  - `src/features/owner/components/court-schedule-editor.tsx`
  - `src/features/owner/components/court-schedule-editor/helpers.ts`
  - `src/features/owner/components/court-schedule-editor/hooks.ts`
  - `src/features/owner/components/court-addon-editor.tsx`
  - `src/features/place-addon/components/place-addon-editor.tsx`
  - `src/features/court-addons/schemas.ts`
  - `src/features/court-addons/helpers.ts`
  - `src/features/place-addon/helpers.ts`
- Admin forms/editors:
  - `src/features/admin/schemas.ts`
  - `src/features/admin/pages/admin-courts-new-page.tsx`
  - `src/features/admin/components/admin-court-edit-form.tsx`

- Remove country/timezone/currency inputs; keep any display text static where needed.

### 5) Mobile owner API compatibility alignment
- Ensure REST mobile owner routes align with removed write fields and read redaction:
  - `src/app/api/mobile/v1/owner/organizations/[organizationId]/venues/route.ts`
  - `src/app/api/mobile/v1/owner/venues/[venueId]/route.ts`
- Keep normalization safe (ignore locale inputs if present), but do not expose locale fields in owner/admin management responses.

## Execution Order (low risk)
1. Add constants and server default enforcement.
2. Update frontend consumers to stop depending on locale read fields (owner/admin).
3. Remove write DTO fields + client payload fields.
4. Apply owner/admin read redaction.
