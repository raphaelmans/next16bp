# Get Started Courts CTA Query-State Fix Plan

## Goal
Ensure the `Courts need updates` card from `/owner/get-started` sends users into the setup query-state flow (`courtId`, `step`, `from=setup`) so the `Edit schedule & pricing` behavior matches the intended setup journey.

## Confirmed Root Cause
- In `src/app/(auth)/owner/get-started/page.tsx`, the `hasActiveCourt` branch currently uses path-only links:
  - `Manage courts` -> `/owner/venues/:placeId/courts`
  - `Edit schedule & pricing` -> `/owner/venues/:placeId/courts/:courtId/schedule`
- This bypasses the setup route query state used elsewhere (`/owner/venues/:placeId/courts/setup?courtId=...&step=schedule`) and explains why no query state appears.

## Recommended Implementation
1. Update the `hasActiveCourt` branch links in `ConfigureVenueCourtsCard` inside `src/app/(auth)/owner/get-started/page.tsx`.
2. Change `Manage courts` to preserve setup origin by appending `?from=setup` to `appRoutes.owner.places.courts.base(placeId)`.
3. Change `Edit schedule & pricing` to use setup query-state destination:
   - Base builder: `appRoutes.owner.places.courts.setup(placeId, courtId, "schedule")`
   - Append setup origin with `&from=setup` (since the setup builder already includes query params).
4. Keep current guards (`placeId` and `courtId`) unchanged so links only render when IDs are present.
5. Do a local consistency check in the same file to ensure no remaining `Edit schedule & pricing` CTA in this card points to the path-only schedule route.

## Critical File To Modify
- `src/app/(auth)/owner/get-started/page.tsx`
