# Plan: Add Amenities Display to Venue Detail Page

## Problem
Amenities are not displayed on the venue detail page (`/venues/[slug]`) despite the data being available from the backend.

## Root Cause
1. Backend returns `amenities` via `PlaceWithDetails` (place.repository.ts:38)
2. Server component uses `placeDetails.amenities` for JSON-LD structured data (page.tsx:53-59)
3. **Missing:** Client hook `usePlaceDetail` does NOT map `amenities` to returned `PlaceDetail` interface (hooks.ts:526-549)
4. **Missing:** No UI component to display amenities on the place detail page

## Implementation

### Step 1: Update PlaceDetail interface and hook
**File:** `src/features/discovery/hooks.ts`

Add `amenities` to the `PlaceDetail` interface (after `contactDetail`):
```ts
amenities: string[];
```

Map amenities in the `usePlaceDetail` hook's select callback (around line 653):
```ts
amenities: response.amenities.map((a) => a.name).filter(Boolean),
```

### Step 2: Create PlaceDetailAmenitiesCard component
**File:** `src/features/discovery/place-detail/components/place-detail-amenities-card.tsx`

Create a card following the existing card pattern (similar to `place-detail-courts-card.tsx`):
- Card wrapper with CardHeader/CardTitle "Amenities"
- CardContent with flex-wrap badges showing amenity names
- Empty state: "No amenities listed" if array empty
- Use Badge component with `variant="secondary"` for consistency

Design following kudoscourts-design-system.md:
- Font: Outfit 600 for title, badges use existing Badge styles
- Radius: Card uses `radius-lg` (12px) via Card component
- Badge pill style: padding 6px 14px, radius 20px (already in Badge)

### Step 3: Add AmenitiesCard to venue detail page
**File:** `src/features/discovery/place-detail/place-detail-page-view.tsx`

Add `<PlaceDetailAmenitiesCard amenities={place.amenities} />` in the two rendering paths:

1. **Booking enabled path:** Add after `<PlaceDetailCourtsCard />` in the left column (line ~176-200 area)
2. **Non-booking path:** Add after `<PlaceDetailContactCard />` in the left column (line ~177 area)

Only render if `place.amenities.length > 0` to avoid empty cards.
