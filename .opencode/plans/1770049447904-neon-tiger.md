# Check: Can org/court owners add amenities? + Plan to enable place-level owner edits

## Current behavior (as implemented)
- Organization owners / court owners do **not** have a way to add or edit amenities for their venues.
- Amenities are only editable through the **admin** curated court flows (admin create + admin update), and are persisted as **place-level** amenities (`place_amenity`).

## Evidence (key code paths)

### Admin can add/edit amenities
- Admin pages include an Amenities checkbox grid and submit `amenities`:
  - `src/app/(admin)/admin/courts/new/page.tsx` (submits `amenities`)
  - `src/features/admin/components/admin-court-edit-form.tsx` (amenities checkbox grid)
- Admin routes are protected by admin session guard:
  - `src/app/(admin)/layout.tsx` uses `requireAdminSession(...)`
  - `src/lib/shared/infra/auth/server-session.ts` redirects if `session.role !== "admin"`
- Admin tRPC router for courts is **admin-only**:
  - `src/lib/modules/court/admin/admin-court.router.ts` uses `adminProcedure` / `adminRateLimitedProcedure`
  - `src/lib/shared/infra/trpc/trpc.ts` enforces `ctx.session.role === "admin"`
- Admin service writes amenities to `place_amenity`:
  - `src/lib/modules/court/services/admin-court.service.ts` (replace-all delete + reinsert)
  - `src/lib/modules/court/repositories/admin-court.repository.ts` (`createAmenity`, `deleteAmenitiesByPlaceId`)
  - Table definition: `src/lib/shared/infra/db/schema/place-amenity.ts`

### Owners cannot add/edit amenities
- Owner place management API does not accept amenities:
  - `src/lib/modules/place/place-management.router.ts` (create/update schemas do not include amenities)
- Owner place form schema + UI do not include amenities:
  - `src/features/owner/schemas.ts` (`placeFormSchema` has no `amenities`)
  - `src/features/owner/components/place-form.tsx` (no amenities section)

## Notes / potential gaps
- There is a `court_amenity` table in TS schema and in the initial SQL migration, but there are no owner-facing endpoints/UI using it.
- No DB RLS policies for amenities are present in repo; writes are controlled at the app layer via admin-only routers.

## Plan: Enable organization owners to manage place amenities (place-level)

### Goal
- Allow organization owners (i.e. users who pass `PlaceManagementService.assertOwner`) to add/edit/clear amenities for their places.
- Scope: **place-level amenities only** (`place_amenity`). No court-level amenities.

### Backend changes
- Extend place management DTOs to accept amenities:
  - Update `src/lib/modules/place/dtos/place.dto.ts`:
    - `CreatePlaceSchema`: add `amenities: z.array(S.place.amenity).optional()`
    - `UpdatePlaceSchema`: add `amenities: z.array(S.place.amenity).optional()`
    - Semantics: `amenities === undefined` means “do not change”; `amenities: []` means “clear all”.
- Persist amenities in the same transaction as other updates:
  - Add write helpers in `src/lib/modules/place/repositories/place.repository.ts` (or a new `place-amenity.repository.ts` if preferred):
    - `deleteAmenitiesByPlaceId(placeId, ctx)`
    - `createAmenities(placeId, names, ctx)` (bulk insert)
    - `replaceAmenities(placeId, names, ctx)` (delete then insert)
  - Update `src/lib/modules/place/services/place-management.service.ts`:
    - `createPlace`: after place creation, if `data.amenities?.length`, insert.
    - `updatePlace`: if `data.amenities` is provided, replace (delete then insert if non-empty).

### Frontend changes (owner)
- Add an Amenities section to the owner place form:
  - Update `src/features/owner/schemas.ts` to include `amenities` in `placeFormSchema` and `defaultPlaceFormValues`.
  - Update `src/features/owner/components/place-form.tsx` to render amenities checkboxes (same UI style as admin).
- Ensure amenities are sent on submit:
  - Update `src/features/owner/hooks.ts` (`usePlaceForm`) to pass `amenities` through to `trpc.placeManagement.create/update`.
- Hydrate existing amenities on edit:
  - Update `src/app/(owner)/owner/places/[placeId]/edit/page.tsx` to set `defaultValues.amenities = placeData.amenities.map((a) => a.name)`.

### Shared amenities list (avoid duplication)
- Create `src/common/amenities.ts` exporting a canonical `AMENITIES` (or `PLACE_AMENITIES`) array.
- Update `src/features/admin/schemas.ts` to import and re-export that list so admin UI remains unchanged.
- Use the same shared list in `src/features/owner/components/place-form.tsx`.
