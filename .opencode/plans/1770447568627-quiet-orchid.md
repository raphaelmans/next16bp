# Plan: Add Pending Tab To My Reservations

## Goal

- Add a new **Pending** tab on player `/reservations` (My Reservations).
- Pending shows reservations that have not reached `CONFIRMED` (e.g. awaiting confirmation/payment).
- Upcoming shows only `CONFIRMED` reservations that are upcoming or in-progress; it must not include pending.
- Update tab badge counts so they match the new bucketing rules.
- Update the existing user story so docs match the new behavior.

## Background (Current State)

- Page: `src/app/(auth)/reservations/page.tsx` renders `ReservationTabs`.
- Tabs UI: `src/features/reservation/components/reservation-tabs.tsx` uses `reservationTabs` from `src/features/reservation/hooks.ts`.
- Bucketing happens client-side in `src/features/reservation/hooks.ts`.
- Current bug: `isUpcomingReservation` includes any non-cancelled status, so pending reservations appear in Upcoming.

Canonical statuses:
- Pending-ish: `CREATED`, `AWAITING_PAYMENT`, `PAYMENT_MARKED_BY_USER`
- Final confirmed: `CONFIRMED`
- Cancelled bucket: `CANCELLED`, `EXPIRED`

## Implementation Plan

1. Update user story (docs)
   - Update `agent-plans/user-stories/06-court-reservation/06-04-player-views-my-reservations-tabs.md`:
     - Tabs become: Upcoming, Pending, Past, Cancelled.
     - Define buckets:
       - Pending: status in {CREATED, AWAITING_PAYMENT, PAYMENT_MARKED_BY_USER}
       - Upcoming: status = CONFIRMED and slotEndTime >= now (or endTime unknown)
       - Past: status = CONFIRMED and slotEndTime < now
       - Cancelled: status in {CANCELLED, EXPIRED}
     - Add AC that Upcoming must not include pending.
   - Update `agent-plans/user-stories/06-court-reservation/06-00-overview.md` story index row for US-06-04 description to mention Pending.

2. Add the new tab key + URL-state support
   - Update `src/features/reservation/hooks.ts`:
     - Change `reservationTabs` to include `"pending"` (recommended order: `upcoming`, `pending`, `past`, `cancelled`).
     - `ReservationTab` + `useReservationsTabs()` will update automatically via the tuple.

3. Define pending/upcoming predicates (fix the bug)
   - Update `src/features/reservation/hooks.ts`:
     - Add `PENDING_STATUSES = new Set(["CREATED","AWAITING_PAYMENT","PAYMENT_MARKED_BY_USER"])`.
     - Add `isPendingReservation(item)`.
     - Update `isUpcomingReservation(item, now)` to require `item.status === "CONFIRMED"` and `endTime === null || endTime >= now`.
     - Keep `isPastReservation` as `CONFIRMED` + `endTime < now`.
     - Keep `isCancelledReservation` as `CANCELLED|EXPIRED`.
     - Make `parseIsoDate` return `null` for invalid dates (so missing slot times don’t crash comparisons).

4. Wire the new tab into list filtering + server-side status prefilter
   - Update `src/features/reservation/hooks.ts`:
     - Update `getStatusFilter(tab)`:
       - `upcoming` -> `"CONFIRMED"`
       - `past` -> `"CONFIRMED"`
       - `pending` / `cancelled` -> `undefined` (needs multi-status)
     - Update the filter switch in `useMyReservations`:
       - `pending`: `isPendingReservation`
       - `upcoming`: `isUpcomingReservation`
       - `past`: `isPastReservation`
       - `cancelled`: `isCancelledReservation`
     - Sorting:
       - Upcoming: startTime asc
       - Pending/Past/Cancelled: startTime desc

5. Update tab badge counts to include Pending
   - Update `useReservationCounts()` in `src/features/reservation/hooks.ts`:
     - Fetch:
       - all (no status)
       - confirmed (`status: "CONFIRMED"`) for upcoming/past
       - cancelled (`status: "CANCELLED"`)
       - expired (`status: "EXPIRED"`)
     - Compute:
       - pending = `all.filter(isPendingReservation)`
       - upcoming = `confirmed.filter(isUpcomingReservation)`
       - past = `confirmed.filter(isPastReservation)`
       - cancelled = cancelled.length + expired.length
     - Return `{ upcoming, pending, past, cancelled }`.

6. Update UI labels + empty state copy
   - Update `src/features/reservation/components/reservation-tabs.tsx`:
     - Add `pending: "Pending"` to `tabLabels`.
   - Update `src/features/reservation/components/reservation-list.tsx`:
     - Add `pending` entry to `emptyStateConfig`.
