# Fix iOS Zoom on Input Fields (min 16px font-size)

## Problem
iOS Safari auto-zooms when focusing inputs with font-size < 16px. Several components use `text-sm` (14px) which triggers this.

## Files to Modify

### 1. Raw inputs using `text-sm` (must fix)
- `src/features/owner/components/booking-studio/mobile-guest-form.tsx` - multiple `<input>` and `<textarea>` with `text-sm`
- `src/features/owner/components/booking-studio/selection-panel-form.tsx` - `<textarea>` with `text-sm`

**Fix:** Change `text-sm` → `text-base` (or remove to inherit 16px)

### 2. UI components with `text-sm` (consider fixing)
- `src/components/ui/input-otp.tsx` - OTP input slots use `text-sm`
- `src/components/ui/input-group.tsx` - addon and text elements use `text-sm` (these are labels, not inputs - may be OK)
- `src/components/ui/select.tsx` - SelectTrigger uses `text-sm`

**Fix:** Change `text-sm` → `text-base` for interactive elements

### 3. Already correct (no change needed)
- `src/components/ui/input.tsx` - uses `text-base` (16px on mobile)
- `src/components/ui/textarea.tsx` - uses `text-base` (16px on mobile)
- `src/components/form/fields/StandardFormInput.tsx` - default size inherits `text-base`
