# Plan: Claim Modal Overflow Fix

## Goal
Increase the Claim Search modal size and eliminate result overflow (horizontal + vertical) by using the existing shadcn `ScrollArea` pattern while preserving current behavior and styling conventions.

## Critical File
- `src/app/(auth)/owner/get-started/page.tsx`

## Implementation Approach
1. Add `ScrollArea` import from `@/components/ui/scroll-area` in `page.tsx`.
2. Update `ClaimSearchDialog` `DialogContent` sizing from `sm:max-w-lg` to a larger responsive width and bounded viewport-safe height (e.g. `sm:max-w-2xl lg:max-w-3xl max-h-[85dvh] overflow-hidden`).
3. Convert the dialog body wrapper (`space-y-4`) into a flex column container (`flex min-h-0 flex-col gap-4`) so the result area can size/scroll correctly.
4. Replace the current results wrapper (`max-h-64 overflow-y-auto ...`) with `ScrollArea` and move list spacing/padding to an inner wrapper, creating one intentional scroll region for results.
5. Tighten result row layout to prevent horizontal overflow with long names/locations:
   - keep button non-shrinking (`shrink-0`),
   - make text column flexible (`min-w-0 flex-1`),
   - keep safe text truncation/wrapping so content stays inside row width.
6. Keep all existing states and flows unchanged: loading spinner, empty state, minimum character hint, claim submit mutation, toast messages, and analytics.

## Style/Pattern Constraints
- Follow shadcn dialog + scroll-area conventions already present in the repo.
- Apply frontend-design improvements only where needed for clarity/usability (spacing, readability), without introducing a new visual system.
