# Availability Studio date-range visual cue

## Goal
Add a clear click affordance to the mobile date-range trigger (`Feb 8 - Feb 14, 2026`) in Availability Studio by adding a trailing caret that rotates when the calendar dialog is open.

## Scope
- Modify only: `src/app/(owner)/owner/bookings/page.tsx`
- Keep existing dialog behavior and state wiring unchanged.

## Implementation plan
1. Update the `lucide-react` import in `src/app/(owner)/owner/bookings/page.tsx` to include `ChevronDown`.
2. In the mobile trigger button inside the `2xl:hidden` section (the button that renders `CalendarIcon` + `weekLabel` and opens `setMobileCalendarOpen(true)`):
   - Keep current button behavior and sizing.
   - Adjust layout classes for icon/text/caret spacing and truncation (`min-w-0`, `gap-*`, label `truncate` where needed).
3. Append a trailing `ChevronDown` icon to that button.
4. Drive caret open-state animation with existing state:
   - Use `mobileCalendarOpen` to conditionally apply `rotate-180`.
   - Add `transition-transform` to smooth the rotation.
5. Add lightweight accessibility attributes to the trigger button:
   - `aria-haspopup="dialog"`
   - `aria-expanded={mobileCalendarOpen}`
   - Mark the caret icon as decorative (`aria-hidden`).

## Style conventions to follow
- Match existing owner/discovery patterns for cues:
  - Small trailing caret (`h-3.5 w-3.5`)
  - Conditional rotation with `cn(...)`
  - Keep visuals subtle and consistent with existing `outline` trigger styling.

## Verification
1. Run `pnpm lint` from repo root.
2. Manual check in Availability Studio (`/owner/bookings`) on mobile-width viewport:
   - Date-range trigger now shows trailing caret.
   - Clicking trigger opens calendar dialog and rotates caret.
   - Closing dialog returns caret to default orientation.
   - `Today` button alignment remains intact and text truncates correctly for long week labels.
3. Keyboard/accessibility smoke check:
   - Trigger remains keyboard-activatable.
   - `aria-expanded` reflects open/closed state.
