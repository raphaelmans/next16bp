# Fix past-time booking in reservation pickers

## Goal
Prevent players from selecting/booking time slots whose start time is already in the past relative to the current time.

## What’s happening (root cause)
- Multiple UI pickers only disable past slots when the parent passes a “now” value.
- Some screens don’t pass it at all (so past hours remain selectable).
- One screen passes a `currentTimeISO` computed once on mount, so if the user keeps the page open it becomes stale.
- `AvailabilityWeekGrid` uses `currentHour` (hour-only), which fails when it’s e.g. 10:30 (10:00 is still treated as not past).
- Backend reservation creation currently has no “start time must be >= now” check, so even if UI is fixed, past bookings can still be created via API.

## Plan

### 1) Add a shared “now” hook (client)
- Add `src/common/hooks/use-now.ts` exporting `useNowMs({ intervalMs })`.
- Default interval: 30s or 60s (hourly slots -> 60s is sufficient; 30s makes the UI feel more responsive around boundaries).

### 2) Make pickers self-enforce “past slot” disabling
Make the components compute “now” internally (and update periodically), so callers don’t need to pass `currentTimeISO/currentHour`.

- Update `src/components/kudos/time-range-picker.tsx`
  - Remove `currentTimeISO` prop (only used by one caller and currently stale there).
  - Use `useNowMs()` and mark a row as past when `Date.parse(slot.startTime) < nowMs`.

- Update `src/components/kudos/availability-week-grid.tsx`
  - Remove `currentHour` prop.
  - Use `useNowMs()` and disable a cell when `slot && Date.parse(slot.startTime) < nowMs`.
  - Keep existing day clamping (`dk < todayDayKey`, `dk > maxDayKey`).

- Update `src/components/kudos/time-slot-picker.tsx`
  - Add “past” disabling in `getSlotState` (or equivalent) using `useNowMs()`.
  - Treat `past` the same as booked/held for `disabled` behavior.

### 3) Remove stale “now” props from callers
- Update `src/app/(public)/places/[placeId]/place-detail-client.tsx`
  - Remove `currentHour` and `currentTimeISO` computations and props passed to pickers.
  - Rely on the picker components’ internal `useNowMs()`.

### 4) Enforce on the backend (prevents bypass)
- Add a new reservation validation error:
  - Update `src/lib/modules/reservation/errors/reservation.errors.ts`
  - Add `ReservationStartTimeInPastError` (code e.g. `RESERVATION_START_TIME_IN_PAST`) including `startTime` and `now` ISO strings.

- Update `src/lib/modules/reservation/services/reservation.service.ts`
  - Add a private `assertStartTimeNotInPast(startTime: Date)`.
  - Call it in:
    - `createReservationForCourt` after parsing `startTime`.
    - `createReservationForAnyCourt` after parsing `startTime`.

- (Optional, depends on product decision) Update `src/lib/modules/reservation/services/reservation-owner.service.ts`
  - Decision: do NOT block backdated owner guest bookings (players only).
