# Plan: Reliably Show Claimed Listing Name

## Goal
Make the pending-claim card on Owner Setup reliably show the claimed listing name, including for existing pending claims where the current name lookup may fail.

## Critical File
- `src/app/(owner-onboarding)/owner/get-started/page.tsx`

## Recommended Approach
1. In `ClaimListingCard`, keep deriving a single pending `CLAIM` request, but make selection deterministic (prefer the latest by `createdAt`) so card name and chat target always refer to the same request.
2. Resolve claimed listing name from `pendingClaim.placeId` via `trpc.place.getById.useQuery` (enabled only when `placeId` exists), and read `placeQuery.data?.place.name`.
3. Replace dependence on `trpc.claimRequest.getById` for name rendering; keep pending card visible immediately and show generic copy until place data is available.
4. Keep `SupportChatSheet` unchanged, still keyed by the selected pending claim `id`.
5. Preserve existing fallback sentence when place lookup fails or returns no name.

## Why This Approach
- Keeps the fix frontend-only and localized to one file.
- Uses a public place lookup by `placeId`, which is less brittle for this display use case.
- Avoids backend contract churn while improving reliability for already-pending claims.

## Edge Cases
- If claim list is loading or no pending claim is found yet, keep current generic pending text.
- If place lookup fails or place has no name, keep current generic pending text.
- If multiple pending claims exist, use one deterministic claim so displayed name and chat request ID always match.

## Verification
1. Run `pnpm lint` from repo root.
2. Manual check in `pnpm dev`:
   - Open `/owner/get-started` with an existing pending claim.
   - Confirm card still shows `Claim pending` / `Under review`.
   - Confirm message includes the claimed listing name.
   - Confirm `Message admin` still opens chat for the same pending request.
   - Confirm fallback generic text appears only when place data is unavailable.
   - Confirm no-pending-claim state remains unchanged.
