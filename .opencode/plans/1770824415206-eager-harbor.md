# Plan: Show Claimed Name in Pending Claim Card

## Goal
Update the Owner Setup pending-claim state to show the name of the listing (court/venue) the user is claiming, while preserving existing behavior and fallbacks.

## Critical File
- `src/app/(auth)/owner/get-started/page.tsx`

## Recommended Approach
1. In `ClaimListingCard`, derive a single `pendingClaim` object (status `PENDING` + `requestType` `CLAIM`) and reuse it for both ID and display data.
2. Use the existing `trpc.claimRequest.getById.useQuery` with `pendingClaim.id` (enabled only when a pending claim exists) to fetch claim details that include `place`.
3. Read the claimed listing name from `pendingClaimDetails.place.name` and render it in the pending message copy.
4. Keep a safe fallback to the current generic sentence when name is unavailable (loading/error/missing place).
5. Keep `SupportChatSheet` behavior unchanged and tied to the same selected `pendingClaim.id`.

## Why This Approach
- Reuses existing API (`claimRequest.getById`) instead of changing backend contracts.
- Limits the change to one page-level file with low risk.
- Keeps current card states and actions intact while improving clarity for the user.

## Edge Cases
- If details are still loading, keep generic pending text until the name arrives.
- If claim details fail to load or place is missing, continue showing generic pending text.
- If multiple pending claims exist, the card continues to use the same selected pending claim for both displayed name and chat button target.

## Verification
1. Run `pnpm lint` from repo root.
2. Manual check in `pnpm dev`:
   - Open Owner Setup with a pending claim.
   - Confirm card still shows `Claim pending` / `Under review`.
   - Confirm message includes the claimed listing name when available.
   - Confirm `Message admin` still opens the chat for that pending request.
   - Confirm no-pending-claim state remains unchanged.
