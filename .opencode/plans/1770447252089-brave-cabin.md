# Plan: Live countdown in reservation status banner

## Goal
Make the payment deadline text inside `AlertTitle` update live (without page refresh) while preserving the current display format: `(Expires in X minutes)`.

## Root cause
`StatusBanner` currently renders `formatRelative(expiresAt)` once per render in `src/features/reservation/components/status-banner.tsx`. Since there is no local timer state in that component, the text only changes when external rerenders happen (polling/manual refresh).

## Recommended approach
Use the existing `useNowMs` hook to trigger lightweight periodic rerenders in `StatusBanner`, and compute the relative label against that ticking "now" value.

This keeps the UI format the same and avoids changes to shared Alert primitives.

## Files to modify
1. `src/features/reservation/components/status-banner.tsx`
2. `src/common/format.ts`

## Planned edits
1. In `src/common/format.ts`, extend `formatRelative` with an optional base-time argument (e.g. `baseDate?: Date | string`) while keeping current default behavior for all existing callers.
2. In `src/features/reservation/components/status-banner.tsx`:
   - Import and use `useNowMs` from `src/common/hooks/use-now.ts`.
   - Tick at minute-level cadence (recommended `30_000` or `60_000` ms).
   - Pass the ticking base time into `formatRelative` for the countdown span so the label recomputes continuously.
   - Keep `AlertTitle` text structure unchanged: `Payment Required` + `(Expires ...)`.
3. Do not modify `src/components/ui/alert.tsx` or `src/features/reservation/components/reservation-expired.tsx` (presentational only for this issue).

## Verification
1. Run `pnpm dev` and open a reservation in `AWAITING_PAYMENT` state.
2. Confirm the `AlertTitle` countdown updates on its own over time (no manual refresh).
3. Confirm format remains the same style (`(Expires in X minutes)`).
4. Confirm other statuses and banner actions (`Message Owner`, `Pay Now`) are unaffected.
5. Run `pnpm lint`.
