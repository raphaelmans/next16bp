# Stream Chat PoC (Provider Adapter + Auth + DM + UI)

## Goal
Ship a minimal, gated Stream Chat proof-of-concept that:
- Uses an adapter/strategy provider layer so we can swap to a self-hosted provider later.
- Mints Stream user tokens server-side and ensures users exist.
- Creates/opens a deterministic 1:1 DM channel for two user IDs.
- Exposes a protected dashboard page that connects Stream Chat React UI.

## Key Decisions
- API shape: use `tRPC chatPoc` (selected) to reuse existing auth context + typed hooks.

## Dependencies
- Add: `stream-chat`, `stream-chat-react`

## Environment Wiring
Update:
- `.env.example`
- `src/lib/env/index.ts`

Add server vars (all optional so builds don’t break when PoC is unused):
- `CHAT_PROVIDER` (default `stream`)
- `CHAT_POC_ENABLED` (default disabled; prod requires `true`)
- `STREAM_CHAT_API_KEY`
- `STREAM_CHAT_API_SECRET`

Implementation notes:
- Parse `CHAT_POC_ENABLED` as `v === "true"` (don’t use `z.coerce.boolean()`; it treats "false" as truthy).
- Keep Stream keys optional in env schema; enforce presence at runtime when provider is `stream`.

## Backend Module: `src/lib/modules/chat/**`
Create a small provider abstraction + service.

Files (proposed):
- `src/lib/modules/chat/types.ts`
  - `ChatProviderId`, `ChatUser`, `ChatAuthResult`, `DmResult`
- `src/lib/modules/chat/providers/chat.provider.ts`
  - `IChatProvider` interface (provider-agnostic operations)
- `src/lib/modules/chat/providers/stream-chat.provider.ts`
  - `StreamChatProvider` implementation using `StreamChat` server client
- `src/lib/modules/chat/factories/chat.factory.ts`
  - `makeChatProvider()` (select by `env.CHAT_PROVIDER`)
  - `makeChatService()`
- `src/lib/modules/chat/services/chat.service.ts`
  - `ensureUser(...)`
  - `mintUserToken(userId)`
  - `getOrCreateDm({ userId, otherUserId })`
- `src/lib/modules/chat/helpers/dm-channel-id.ts`
  - Deterministic channel id helper (hash-based)

Deterministic DM channel id
- Don’t use `dm:${a}:${b}`:
  - `:` is risky/invalid in Stream channel IDs, and UUID-based IDs will exceed common 64-char limits.
- Use a stable hash: `dm-<sha256(min+":"+max).slice(0, 32)>`.

Stream provider behaviors (via `stream-chat` docs)
- Server client: `new StreamChat(apiKey, apiSecret, { disableCache: true })`
- Ensure user(s): `client.upsertUsers([{ id, name?, image? }, ...])`
- Mint token: `client.createToken(userId)`
- Create channel: `client.channel("messaging", channelId, { created_by_id: userId, members })` + `await channel.create()`

## API Layer (Recommended): tRPC
Create:
- `src/lib/modules/chat/chat-poc.router.ts`
  - `getAuth` (protected)
    - Guard: if prod and `env.CHAT_POC_ENABLED !== true` => throw tRPC NOT_FOUND with `NotFoundError` cause
    - Ensure current user exists in chat provider
    - Return `{ apiKey, user, token }`
  - `getOrCreateDm` (protected)
    - Input: `{ otherUserId: string }` (optionally `z.string().uuid()`)
    - Guard: same as above
    - Ensure both users exist + create DM channel
    - Return `{ channelId, memberIds }`

Wire into root:
- `src/lib/shared/infra/trpc/root.ts` add `chatPoc: chatPocRouter`

## UI
Route:
- `src/app/(protected)/dashboard/chat-poc/page.tsx`
  - Server component
  - Prod gate: if `process.env.NODE_ENV === "production" && env.CHAT_POC_ENABLED !== true` => `notFound()`
  - Render `<ChatPocClient />`

Scoped CSS import:
- `src/app/(protected)/dashboard/chat-poc/layout.tsx`
  - Import Stream UI CSS: `import "stream-chat-react/dist/css/v2/index.css";`
  - Render `{children}`

Client component:
- `src/features/chat/components/ChatPocClient.tsx`
  - `"use client"`
  - `trpc.chatPoc.getAuth.useQuery()`
  - `useCreateChatClient({ apiKey, tokenOrProvider: token, userData: user })`
  - UI: input `otherUserId` + button
  - On submit: `trpc.chatPoc.getOrCreateDm.useMutation()`; then `client.channel("messaging", channelId, { members: memberIds }).watch()`; render `<Channel>`
  - Render Stream components: `Chat`, `Channel`, `Window`, `ChannelHeader`, `MessageList`, `MessageInput`, `Thread`
