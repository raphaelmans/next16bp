# Fix Week Timeline Range + Grey Out Closed Hours

## Problem
- Owner week timeline grid is truncated when court hours differ by day.
  - Example: Mon-Sat 08:00-20:00, Sun 08:00-13:00.
  - Current week view uses the *selected* day’s hours to build `hours[]`, so if the selected day is Sunday the whole week renders/clamps to 08:00-13:00.

## Root Cause
- `src/app/(owner)/owner/places/[placeId]/courts/[courtId]/availability/page.tsx` computes `timelineRange` from `parseTimelineRange(courtHours, selectedDayOfWeek)` even in week view, then uses that range for:
  - the vertical time axis (`hours`)
  - clamping/rendering blocks/reservations per day
- `WeekDayColumn` selection currently only considers blocks/reservations; it does not consider court hours windows, so once the axis is widened, closed hours and gaps must be explicitly disabled/greyed.

## Goals
1) Week view axis uses the union of court hours across the visible week:
   - `startHour = min(start)` and `endHour = max(end)` across visible days.
   - Days with no windows (closed days) must NOT widen the union.
   - If *all* visible days have no windows, fall back to defaults.
2) Cells outside that day’s court hours windows (and gaps between multiple windows) are:
   - visually greyed out
   - non-interactive (range selection + click)
   - (nice-to-have) not droppable for DnD-kit
3) Cover scenario variety: closed day, multiple windows with gaps, end at 24:00, non-hour-aligned windows given Snap=60m.

## Plan

### 1) Add a small court-hours helper (re-used by both owner pages)
Add `src/features/owner/components/booking-studio/court-hours.ts`:
- Types:
  - `CourtHoursWindow = { dayOfWeek: number; startMinute: number; endMinute: number }`
- Helpers:
  - `getDayOfWeekForDayKey(dayKey, timeZone)` (uses `getZonedDayRangeFromDayKey` + `getZonedDate`)
  - `getWindowsForDayOfWeek(windows, dayOfWeek)`
  - `getTimelineRangeForDayOrNull(windows, dayOfWeek)`
    - returns `null` when no windows
    - otherwise returns `{ startHour, endHour }` using floor/ceil of min/max minutes
  - `getTimelineRangeForWeek(windows, weekDayKeys, timeZone)`
    - gathers per-day ranges via `getTimelineRangeForDayOrNull`
    - ignores `null` (closed days)
    - falls back to `DEFAULT_START_HOUR/DEFAULT_END_HOUR` only if all are `null`
  - `buildOpenCellIndexSet({ windowsForDay, axisStartHour, cellCount, snapMinutes: 60 })`
    - for each hour-cell index i, mark open only if a window fully covers `[slotStart, slotEnd)`
    - handles gaps + endMinute=1440 correctly

### 2) Fix week axis in owner court availability page
Update `src/app/(owner)/owner/places/[placeId]/courts/[courtId]/availability/page.tsx`:
- Keep day view range: `selectedTimelineRange = parseTimelineRange(courtHours, selectedDayOfWeek)`.
- Replace `timelineRange` with:
  - week view: `getTimelineRangeForWeek(courtHours, weekDayKeys, placeTimeZone)`
  - day view: `selectedTimelineRange`
- Keep `hours`, `timelineStartMinute`, `timelineEndMinute` derived from the selected `timelineRange`.

### 3) Disable/grey out closed hours in selection configs

Update `src/features/owner/components/booking-studio/week-day-column.tsx`:
- Add prop: `courtHoursWindows?: CourtHoursWindow[]`.
- In `selectionConfig`, compute `openCellIndices` for this `dayKey` using:
  - `dayOfWeek = getDayOfWeekForDayKey(dayKey, timeZone)`
  - `dayWindows = getWindowsForDayOfWeek(courtHoursWindows, dayOfWeek)`
  - `openCellIndices = buildOpenCellIndexSet({ windowsForDay: dayWindows, axisStartHour: hours[0] ?? 0, cellCount: hours.length, snapMinutes: 60 })`
- Treat a cell as available only if:
  - it is within bounds
  - it is in `openCellIndices`
  - it is not blocked by an existing block/reservation
- Ensure `computeRange`/`clampToContiguous` stop at either blocked OR closed indices, so selection cannot cross gaps.

Update `src/app/(owner)/owner/places/[placeId]/courts/[courtId]/availability/page.tsx` (day view):
- Extend `daySelectionConfig` to also enforce court hours windows (same open-cell logic), so mid-day gaps and closed days are non-interactive.

### 4) Grey styling + prevent droppable on unavailable cells
Update `src/features/owner/components/booking-studio/selectable-timeline-row.tsx`:
- Compute `isCellOpen = isCellAvailable(cellIndex)` from the range-selection config.
- Define `isInteractive = !disabled && isCellOpen`.
- Pass `disabled: !isInteractive` to `useDroppable`.
- Add a muted/grey background style when `!isInteractive` (and remove hover affordances), while keeping current styles for interactive cells.

### 5) Keep the other owner week grid consistent
Update `src/app/(owner)/owner/bookings/page.tsx`:
- Replace the existing week union logic (which uses `parseTimelineRange` and can widen on closed days) with `getTimelineRangeForWeek(...)`.
- Pass `courtHoursWindows` into `WeekDayColumn`.
- Update its day-view selection config to also enforce open windows (same as court availability page).

## Scenario Coverage (Acceptance Checks)
- Different end times per day: Sunday shorter no longer truncates Mon-Sat.
- Different start times per day: earliest start defines axis.
- Closed day (no windows): does not widen axis; entire column is grey + not selectable.
- Split windows with a gap: gap rows are grey + selection cannot cross the gap.
- End at 24:00 (endMinute=1440): axis can reach `endHour=24`; last selectable slot is 23:00-24:00.
- Non-hour-aligned windows with Snap=60m: only full-hour-covered cells are selectable.
