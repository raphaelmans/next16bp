
# Docs: Migration + Component Placement Guardrails

## Context
- The client conventions migration plan targets `src/app/(public)/places/[placeId]/place-detail-client.tsx` for refactor.
- In the codebase, that file still exists (and is large), but core logic was extracted into the discovery feature (`src/features/discovery/hooks.ts`, `src/features/discovery/helpers.ts`) and it uses `src/common/*` utilities.
- The migration docs mark the “Public place detail refactor” as completed, but `agent-plans/80-client-conventions-migration/client-conventions-migration-client-dev1-checklist.md` still has the corresponding checkbox unchecked, which makes it look like the file was never migrated.
- Separately, `CLAUDE.md` / `AGENTS.md` are not strict enough about file placement: we still see new React component modules created directly under `src/app/` (example: `src/app/home-page-client.tsx`).

## Goal
- Make docs consistent and high-signal so future work follows the intended folder structure.

## Scope
- Docs only (no functional code changes).

## Planned Changes
1. Reconcile place detail migration docs
   - Update `agent-plans/80-client-conventions-migration/client-conventions-migration-client-dev1-checklist.md`
     - Mark “Split place detail into sections” as complete.
     - Add a short note clarifying what “done” means here: logic moved to `src/features/discovery/*`; the route-level client file remains as the orchestration layer.

2. Strictly document component placement rules for future commits
   - Update `CLAUDE.md` and `AGENTS.md` (keep the non-generated sections 1:1 as required by `guides/context.md`).
   - Add an explicit rule under `## Conventions`:

     - `Component placement (MUST)`: `src/app/**` is route entrypoints only (`page.tsx`, `layout.tsx`, `loading.tsx`, `error.tsx`, `not-found.tsx`, `route.ts`, etc.). Do not add new component modules under `src/app/**` (treat existing ones as legacy).
     - Put feature UI in `src/features/<feature>/components/**`, shared UI in `src/components/**`, and shared helpers in `src/common/**` (reference: `@guides/client/core/folder-structure.md`).
