# Owner Setup Courts Return Redirect Plan

## Goal
Make the "Manage courts" and "Manage availability" paths from owner setup behave like other setup steps by returning to `/owner/get-started` after successful **schedule save** when the flow originated from setup.

## Confirmed scope
- Redirect trigger: **schedule save only** (confirmed).
- Do not redirect after other actions (availability blocks, guest bookings, deactivate, edit-details-only save).

## Root cause
- In `src/app/(auth)/owner/get-started/page.tsx`, the "Courts configured" complete-card links do not carry setup origin (`from=setup`).
- Downstream court pages therefore cannot detect setup origin and keep default in-page behavior.

## Implementation

### 1) Add setup origin on hub entry links
Modify `src/app/(auth)/owner/get-started/page.tsx`:
- Append `?from=setup` to:
  - "Manage courts" (`appRoutes.owner.places.courts.base(placeId)`)
  - "Manage availability" (`appRoutes.owner.places.courts.availability(placeId, courtId)`)

### 2) Preserve setup origin through courts-management navigation
Modify `src/app/(owner)/owner/places/[placeId]/courts/page.tsx`:
- Read `from` query and derive `isFromSetup`.
- When setup-origin, append `from=setup` to the "Add Court" link.
- Pass setup-origin state into `CourtsTable`.

Modify `src/features/owner/components/courts-table.tsx`:
- Add optional `fromSetup?: boolean` prop.
- When `fromSetup` is true, append `from=setup` to the "Schedule & Pricing" destination (`appRoutes.owner.places.courts.setup(...)`).
- Keep edit/deactivate/bookings behavior unchanged.

### 3) Redirect after schedule save for setup-origin flows
Modify `src/app/(owner)/owner/places/[placeId]/courts/setup/page.tsx`:
- Reuse existing `from` query handling.
- Update `CourtScheduleEditor` `onSaved` behavior:
  - `from=setup` -> `router.push(appRoutes.owner.getStarted)`
  - otherwise keep current step progression (`goToStep("publish")`).

Modify `src/app/(owner)/owner/places/[placeId]/courts/[courtId]/availability/page.tsx`:
- Read `from` query and propagate it to the "Edit schedule" link.

Modify `src/app/(owner)/owner/places/[placeId]/courts/[courtId]/schedule/page.tsx`:
- Read `from` query.
- Pass `onSaved` callback to `CourtScheduleEditor` that redirects to `appRoutes.owner.getStarted` only when `from=setup`.

## Edge cases / safety
- Missing or non-`setup` `from` value: preserve existing behavior.
- Direct navigation without query: unchanged.
- Only accept literal `from=setup`; no arbitrary return URLs.
- Ensure query appending works for links that already include query strings.

## Critical files
- `src/app/(auth)/owner/get-started/page.tsx`
- `src/app/(owner)/owner/places/[placeId]/courts/page.tsx`
- `src/features/owner/components/courts-table.tsx`
- `src/app/(owner)/owner/places/[placeId]/courts/setup/page.tsx`
- `src/app/(owner)/owner/places/[placeId]/courts/[courtId]/availability/page.tsx`
- `src/app/(owner)/owner/places/[placeId]/courts/[courtId]/schedule/page.tsx`

## Verification
1. From `/owner/get-started` (Courts configured card) -> **Manage courts** -> Add Court -> save schedule -> confirm redirect to `/owner/get-started`.
2. From `/owner/get-started` -> **Manage courts** -> existing court "Schedule & Pricing" -> save schedule -> confirm redirect to `/owner/get-started`.
3. From `/owner/get-started` -> **Manage availability** -> "Edit schedule" -> save schedule -> confirm redirect to `/owner/get-started`.
4. In setup-origin availability page, submit block/guest actions -> confirm user remains on availability page.
5. Open courts/availability/schedule directly without `from=setup` -> confirm no forced redirect to setup hub.
6. Run `pnpm lint`.
