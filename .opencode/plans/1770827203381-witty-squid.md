# Plan: Add Refresh Button to Admin Claims

## Goal
Add a refresh button to `/admin/claims` that matches the existing admin refresh pattern used on `/admin/verification`.

## Recommended Implementation

### File to modify
- `src/app/(admin)/admin/claims/page.tsx`

### Changes in that file
1. Update imports:
   - Add `RefreshCw` from `lucide-react`.
   - Add `cn` from `@/lib/utils`.
2. Update `useClaims(...)` destructuring:
   - Keep existing `data` and `isLoading`.
   - Also destructure `isFetching` and `refetch`.
3. Add a refresh button in the page header area (same visual/behavior pattern as verification page):
   - Use `Button` with `variant="outline"` and `size="sm"`.
   - `onClick={() => void refetch()}`.
   - `disabled={isFetching}`.
   - Add `RefreshCw` icon with conditional spin class: `cn("mr-2 h-4 w-4", isFetching && "animate-spin")`.
   - Button label toggles between `"Refreshing"` and `"Refresh"` based on `isFetching`.

## Scope and Constraints
- Scope is list refresh only (user-confirmed): do not add counts/badge refetch wiring.
- Keep change minimal and localized to one route file.
- Preserve existing loading, filtering, search, and pagination behavior.

## Verification
1. Run `pnpm lint`.
2. Run `pnpm dev`, open `/admin/claims`.
3. Confirm refresh button appears in header and matches admin verification button style.
4. Click refresh and verify:
   - Claims query refetches with current filters/search/page.
   - Button disables while fetching.
   - Icon spins and text changes to `Refreshing` during fetch.
   - Button returns to normal state after completion.
5. Confirm counts/badges are not explicitly refreshed by this button (expected scope).
