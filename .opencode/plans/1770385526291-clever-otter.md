# Plan: Owner Sidebar Header Regression Fix

## Objective
Fix the owner sidebar header regression by restoring a **single header row** and showing the **organization name** as the primary header label, while keeping portal switching reusable and non-blocking.

## Root Cause
- `OwnerSidebar` now renders both:
  - `PortalSwitcher variant="sidebar"` and
  - the existing organization switcher row.
  This creates two stacked headers.
- `PortalSwitcher` sidebar variant hardcodes subtitle to `KudosCourts`, so owner header branding no longer reflects organization context.

## Confirmed User Intent
- Only one owner header row.
- Header should show organization context (e.g., `Rethndr`), not `KudosCourts`.
- Reusable switcher architecture must stay.

## Recommended Implementation
1. Consolidate owner header to one trigger row
   - Update `src/features/owner/components/owner-sidebar.tsx`:
     - remove `PortalSwitcher variant="sidebar"` from `SidebarHeader`
     - keep existing organization dropdown trigger row as the single header UI

2. Keep portal switching reusable in the same owner dropdown
   - In owner header dropdown content, render `PortalSwitcher variant="menu-items" isOwner`.
   - Add a separator between portal options and organization options.
   - Keep organization switching items (`?org=<id>`) unchanged.

3. Ensure dropdown is usable even with one organization
   - Owner header dropdown should still open when `currentOrganization` exists, even if `organizations.length === 1`.
   - Show portal switch items in all cases; org-switch list only when there are multiple organizations.

4. Preserve non-blocking switch behavior
   - No changes to portal persistence behavior in `PortalSwitcher`.
   - Continue immediate navigation + fire-and-forget `setDefaultPortal` for player/owner.

## Files to Modify
- `src/features/owner/components/owner-sidebar.tsx`

## Verification
1. Static check
   - Run `pnpm lint`

2. Manual owner checks (`pnpm dev`)
   - Owner sidebar shows one header row only.
   - Header title shows current organization name.
   - Header dropdown includes portal switch options (Player/Owner/Admin where applicable).
   - If multiple organizations, org-switch options are still present and working.
   - Portal switch remains non-blocking (navigation not delayed by mutation).
