# Plan: Portal Switcher Hover Contrast A11y Fix

## Objective
Fix portal switch dropdown hover/active contrast to improve readability and accessibility, while keeping current design tokens and reusable switcher architecture.

## Root Cause
- In `PortalSwitcher` menu-items mode, the current/selected item uses `bg-accent` without paired foreground color, so text/icons can render with low contrast.
- In shared `DropdownMenuItem` styles, highlight colors are tied to `focus:*` only; pointer-hover/highlight states can appear inconsistent against accent backgrounds.

## Confirmed User Intent
- Improve hover contrast in the owner sidebar dropdown (and matching switch surfaces).
- Keep styling aligned with existing theme and frontend design system.

## Recommended Implementation
1. Fix selected-state contrast in reusable portal switcher
   - Update `src/components/layout/portal-switcher.tsx` (menu-items variant):
     - change selected class from `bg-accent` to `bg-accent text-accent-foreground`.
   - This ensures active portal rows always use a paired foreground token.

2. Normalize highlighted-state contrast in dropdown primitive
   - Update `src/components/ui/dropdown-menu.tsx` (`DropdownMenuItem` base classes):
     - add `data-[highlighted]:bg-accent data-[highlighted]:text-accent-foreground`
     - keep existing `focus:bg-accent focus:text-accent-foreground`
   - This aligns keyboard and pointer highlight behavior and prevents mixed contrast outcomes.

3. Keep scope minimal to reduce regressions
   - Do not alter theme tokens or global palette.
   - Do not change dropdown structure or item order.
   - Do not modify checkbox/radio/submenu item styles in this pass.

## Files to Modify
- `src/components/layout/portal-switcher.tsx`
- `src/components/ui/dropdown-menu.tsx`

## Verification
1. Static check
   - Run `pnpm lint`

2. Manual UI/a11y checks (`pnpm dev`)
   - Owner sidebar org dropdown: active and hovered portal rows are clearly readable.
   - Owner/admin/player menu-based switchers: hover and active colors are consistent.
   - Keyboard navigation: highlighted row matches hover contrast (`ArrowDown` + `Enter`).
   - Quick visual check in both light/dark themes (if available) for sufficient contrast.
