# Make Logo Redirect Follow Persisted Portal

## Goal
Ensure authenticated logo clicks no longer hardcode `/home`; they should respect the last persisted portal preference (`player` vs `owner`).

## Recommended approach
1. Reuse the existing server-side redirect resolver by routing logo clicks to `appRoutes.postLogin.base` (`/post-login`) instead of `appRoutes.home.base`.
2. Keep portal decision logic centralized in `src/app/(auth)/post-login/page.tsx` (already resolves based on `user_preferences.default_portal` + ownership check).
3. Update authenticated navbar logo links to point to `/post-login`:
   - `src/components/layout/player-navbar.tsx`
   - `src/features/owner/components/owner-navbar.tsx`
   - `src/features/admin/components/admin-navbar.tsx` (included per confirmed “everywhere” scope)
4. Do not duplicate preference logic in client components; navbars remain simple links.

## Behavior after change
- `defaultPortal = "player"` -> logo lands on `/home`.
- `defaultPortal = "owner"` and user owns an organization -> logo lands on `/owner`.
- `defaultPortal = "owner"` and user has no organization -> logo lands on `/owner/get-started`.

## Critical files
- `src/components/layout/player-navbar.tsx`
- `src/features/owner/components/owner-navbar.tsx`
- `src/features/admin/components/admin-navbar.tsx`
- `src/app/(auth)/post-login/page.tsx` (read-only reference; no logic change expected)

## Verification
1. Run `pnpm dev` and log in with a user that can switch portals.
2. Set portal preference to `player` via portal switcher; click logo from player/owner/admin navbars; verify destination is `/home`.
3. Set portal preference to `owner` for a user with an organization; click logo from player/owner/admin navbars; verify destination is `/owner`.
4. Test owner preference for a user without an organization; verify logo lands on `/owner/get-started`.
5. Confirm unauthenticated/public logo behavior is unchanged.
6. Run `pnpm lint`.
