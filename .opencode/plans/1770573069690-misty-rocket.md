# Remove /post-login Flicker While Keeping Role-Aware Logo Redirect

## Goal
Keep logo clicks role-aware (`player` vs `owner`) **without showing intermediate `/post-login` UI**.

## Why flicker happens
- Logo links now go to `/post-login` from authenticated navbars.
- `/post-login` currently lives at `src/app/(auth)/post-login/page.tsx`.
- `(auth)` layout (`src/app/(auth)/layout.tsx`) wraps protected routes with `PlayerShell`, so shell UI can appear briefly before redirect resolves.
- Next.js docs indicate `redirect()` in streaming contexts can become meta/client redirect behavior; pre-render redirects should use Proxy or non-UI server handlers.

## Recommended implementation
1. Convert `/post-login` from page route to route handler:
   - Remove `src/app/(auth)/post-login/page.tsx`.
   - Add `src/app/(auth)/post-login/route.ts` with `GET` handler that computes destination and returns redirect response.
2. Migrate existing resolver logic as-is into route handler (single source of truth):
   - `requireSession(pathname)`
   - `makeUserPreferenceService().findByUserId(session.userId)`
   - `makeOrganizationRepository().findByOwnerId(session.userId)`
   - fallback:
     - owner + has org -> `appRoutes.owner.base`
     - owner + no org -> `appRoutes.owner.getStarted`
     - player -> `appRoutes.home.base`
3. Preserve redirect sanitization using `getSafeRedirectPath` with:
   - `fallback` from resolved portal destination
   - `disallowRoutes: ["guest"]`
   - `disallowPathname: appRoutes.postLogin.base`
4. Keep navbar links unchanged (already point to `appRoutes.postLogin.base`):
   - `src/components/layout/player-navbar.tsx`
   - `src/features/owner/components/owner-navbar.tsx`
   - `src/features/admin/components/admin-navbar.tsx`

## Notes from docs to honor
- Route handlers and pages cannot coexist at the same route segment (so `page.tsx` must be removed when adding `route.ts`).
- This approach keeps redirect on server response path and avoids rendering shell UI for `/post-login`.

## Critical files
- `src/app/(auth)/post-login/page.tsx` (delete)
- `src/app/(auth)/post-login/route.ts` (new)
- `src/components/layout/player-navbar.tsx` (already updated; verify no further change)
- `src/features/owner/components/owner-navbar.tsx` (already updated; verify no further change)
- `src/features/admin/components/admin-navbar.tsx` (already updated; verify no further change)

## Verification
1. Open DevTools Network, click logo while authenticated, confirm `/post-login` returns redirect response (3xx) and no shell paint is visible.
2. With `defaultPortal = player`, logo lands on `/home` from player/owner/admin navbars.
3. With `defaultPortal = owner` and org exists, logo lands on `/owner`.
4. With `defaultPortal = owner` and no org, logo lands on `/owner/get-started`.
5. Security checks:
   - `/post-login?redirect=https://evil.com` falls back safely.
   - `/post-login?redirect=/login` (guest route) falls back safely.
   - `/post-login?redirect=/post-login` avoids loop.
6. Run `pnpm lint`.
