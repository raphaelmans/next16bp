### Bug 3: Range selection highlight flicker (two-click selection)

Root cause (current)

- `RangeSelectionStore` clears `{anchorIdx, hoverIdx}` immediately after calling `config.commitRange(...)`, but the store’s own `committedRange` only updates later via provider `useEffect` syncing props.
- This creates a transient render where `activeRange` becomes `null`, so cell highlight disappears.

Client / Frontend

- [ ] Update `src/components/kudos/range-selection/range-selection-store.ts` so commits update the store’s `committedRange` immediately (before clearing drag state).
  - Implement an internal helper like `commit(range)` that does:
    - `set({ committedRange: range, suppressClick: true, ... })`
    - then calls `config.commitRange(range.startIdx, range.endIdx)`.
  - Apply this to both `pointerUp()` and `click()` paths.
- [ ] Update `src/components/kudos/range-selection/range-selection-provider.tsx` to sync `committedRange` with `useLayoutEffect` (instead of `useEffect`) so external clears/changes don’t paint stale selection.
- [ ] Manual verify: start click keeps a visible pending highlight; second click extends without blinking.
