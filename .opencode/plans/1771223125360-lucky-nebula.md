# Venue Owner QR Codes (Owner Venues)

## Goal

- Owners can generate a QR code for each venue that opens the public venue detail route (`/venues/[placeId]`).
- Owners can download the QR (PNG + SVG) and share it (Web Share API when available; fallback to copy link).

## Key Repo Context

- Owner venues page is an alias:
  - `src/app/(owner)/owner/venues/page.tsx` re-exports `src/app/(owner)/owner/places/page.tsx`.
  - Real UI is a client component and renders a grid of venue cards.
- Public venue detail is an alias:
  - `src/app/(public)/venues/[placeId]/page.tsx` re-exports `src/app/(public)/places/[placeId]/page.tsx`.
  - Canonical public path is `/venues/{slugOrId}` (slug preferred; UUID redirects to slug when present).
- Canonical route builder: `src/common/app-routes.ts` -> `appRoutes.places.detail(slugOrId)`.
- Clipboard helper + toasts: `src/common/utils/clipboard.ts` (`copyToClipboard`).
- Optional base URL: `src/lib/env/index.ts` (`env.NEXT_PUBLIC_APP_URL`), but it may be intentionally undefined in dev; fallback to `window.location.origin`.

## Package Choice (Context7)

Candidates surfaced via Context7:

- `react-qr-code` (`/rosskhanas/react-qr-code`) - simple SVG rendering.
- `next-qrcode` (`/bunlong/next-qrcode`) - Canvas/SVG/Image via hook.
- `qrcode.react` (`/zpao/qrcode.react`) - React components for Canvas + SVG.

Recommended: `qrcode.react`

- Provides both `QRCodeCanvas` and `QRCodeSVG`, which makes "Download PNG" (canvas export) and "Download SVG" straightforward.
- Works cleanly inside an App Router client component without introducing Next-specific hooks.

## Implementation Plan

### 1) Add dependency

- Add `qrcode.react` to `package.json` (and update `pnpm-lock.yaml`).

### 2) Create QR dialog component (feature-scoped)

- Add `src/features/owner/components/venue-qr-code-dialog.tsx` (client component)
  - Props:
    - `venueName: string`
    - `venueSlugOrId: string` (use `place.slug ?? place.id`)
  - Compute:
    - `publicPath = appRoutes.places.detail(venueSlugOrId)` (always `/venues/...`)
    - `baseUrl = env.NEXT_PUBLIC_APP_URL ?? (mounted ? window.location.origin : "")`
    - `publicUrl = baseUrl ? new URL(publicPath, baseUrl).toString() : publicPath`
    - Guard `window` access via a mounted flag (`useEffect`) to avoid SSR crashes.
  - UI:
    - Use shadcn `Dialog` (`src/components/ui/dialog.tsx`).
    - Trigger: `Button variant="ghost" size="sm"` labeled "QR Code" (optionally with lucide `QrCode` icon).
    - Dialog content: title, description, QR, and action buttons.
  - QR rendering:
    - Render `QRCodeCanvas` for on-screen display + PNG export.
    - Render `QRCodeSVG` hidden for SVG export.
    - Use `marginSize={4}`, `level="M"`, `bgColor="#FFFFFF"`, `fgColor="#000000"`, and a `title` for accessibility.
  - Actions:
    - Copy link: `copyToClipboard(publicUrl, "Venue link")`.
    - Download PNG:
      - Locate the rendered `<canvas>` (via wrapper ref + `querySelector("canvas")`).
      - Use `canvas.toBlob(...)` (fallback to `toDataURL` when needed).
      - Trigger `<a download>`; if download fails on iOS/Safari, fall back to opening the image in a new tab.
    - Download SVG:
      - Locate `<svg>`; `XMLSerializer().serializeToString(svg)` -> Blob -> `<a download>`.
    - Share:
      - If `navigator.share` exists: share `{ title, url: publicUrl }`.
      - If file sharing is supported (`navigator.canShare({ files })`), also share the PNG file.
      - Else fallback to `copyToClipboard` with a toast.

### 3) Export from owner components barrel

- Update `src/features/owner/components/index.ts` to export `VenueQrCodeDialog`.

### 4) Wire into owner venues list

- Update `src/app/(owner)/owner/places/page.tsx`:
  - Import `VenueQrCodeDialog` from `@/features/owner/components`.
  - In each card action row (near "View Public Page"), render:
    - `<VenueQrCodeDialog venueName={place.name} venueSlugOrId={place.slug ?? place.id} />`

## Edge Cases / Non-Goals

- If a venue has no slug, encode the UUID; public page will redirect to slug later if/when it exists.
- If `NEXT_PUBLIC_APP_URL` is unset, generate from `window.location.origin` (works in staging/prod and local dev).
- No backend changes required (pure UI enhancement).

- `pnpm lint`

  - Scan QR with phone camera: lands on `/venues/{slugOrId}`.
  - Download PNG and SVG; verify both are scannable.
  - Share button:
