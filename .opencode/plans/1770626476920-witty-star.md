# Plan: Remove chat inbox search and keep right-aligned refresh

## Goal
Prevent mobile keyboard from opening when the floating chat widget is tapped by removing inbox search UI for now, while keeping the refresh action visible on the right (`justify-end`).

## Recommended implementation
1. **Remove reservation inbox search UI and logic** in `src/features/chat/components/chat-widget/reservation-inbox-widget.tsx`.
   - Delete `Search` icon and `Input` imports.
   - Delete `query` state and `filteredReservationIds` memo.
   - Update sorting to use `reservationIds` directly.
   - Replace the list header row from search+refresh to refresh-only with `className="flex justify-end"`.

2. **Clean reservation config contract and callers**.
   - In `src/features/chat/components/chat-widget/reservation-inbox-widget.tsx`, remove `ui.searchPlaceholder` from `ReservationInboxWidgetConfig`.
   - Remove `searchPlaceholder` from:
     - `src/features/chat/components/chat-widget/player-chat-widget.tsx`
     - `src/features/chat/components/chat-widget/owner-chat-widget.tsx`

3. **Apply the same temporary no-search behavior to support inbox** for consistency and to avoid the same mobile focus issue in admin support chat.
   - In `src/features/support-chat/components/support-inbox-widget.tsx`:
     - Delete `Search` icon and `Input` imports.
     - Delete `query` state and `filteredChannels` memo.
     - Derive `activeChannels` / `archivedChannels` directly from `supportChannels`.
     - Replace list header row with refresh-only and `className="flex justify-end"`.

4. **Do not change Sheet autofocus behavior** right now.
   - No edits in `src/features/chat/components/inbox-shell/inbox-floating-sheet.tsx` or `src/components/ui/sheet.tsx` for this iteration.
   - This keeps scope focused on the requested temporary removal.

## Critical files to modify
