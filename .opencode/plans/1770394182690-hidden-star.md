   - Keep manual refresh as explicit reconciliation tool.
   - Add a simple PR checklist item: every reservation mutation must declare impacted queries.

4. Add a Future Migration Plan section (reservation notifications)
   - Document staged transition (evolutionary, no big-bang):

     - Phase 1: define reservation event schema/types

     - Phase 3: lower polling frequency behind flags once reliability is proven

   - Explicitly keep polling fallback in architecture contract.
