# OG image quick fix plan (black overlay text near logo)

## Objective
Remove the unexpected black text (`KudosCourts logo`) appearing near the logo in generated OG images by applying a minimal, low-risk fix.

## Confirmed root cause
- The inline SVG logos used in OG/Twitter image generators include `<title>KudosCourts logo</title>`.
- In `next/og` rendering, this SVG title text can appear visually as black overlay text.

## Recommended implementation
1. Remove the SVG `<title>` node from all OG/Twitter logo SVGs used by image generation routes.
2. Keep existing text color styling as-is (already white/near-white); no color-system changes needed for this bug.

## Critical files to modify
- `src/app/opengraph-image.tsx`
- `src/app/twitter-image.tsx`
- `src/app/(public)/list-your-venue/opengraph-image.tsx`
- `src/app/(public)/courts/[id]/opengraph-image.tsx`

## Verification
1. Start dev server (`pnpm dev`) and open image endpoints directly:
   - `/opengraph-image`
   - `/twitter-image`
   - `/list-your-venue/opengraph-image`
   - `/courts/<existing-id-or-slug>/opengraph-image`
2. Confirm the black text `KudosCourts logo` no longer appears near the logo in all outputs.
3. Confirm brand/title/tagline text remains white/near-white and layout remains unchanged.
4. Run `pnpm lint` to validate formatting/lint rules.
