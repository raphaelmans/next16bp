# Plan: Fix Owner Reservations Sidebar Active Count

## Final Scope (confirmed)
- Implement a **single badge** on `Owner > Reservations` in the sidebar.
- Badge value must match Reservations page **Inbox > All** count.
- Statuses included in this count: `CREATED`, `AWAITING_PAYMENT`, `PAYMENT_MARKED_BY_USER`.

## Implementation Approach
1. Align backend pending-count semantics with Inbox
   - Update owner reservation service count query to include `AWAITING_PAYMENT`.
   - Keep current `getPendingCount` endpoint/hook path (minimal blast radius, no new endpoint).

2. Render count badge on owner sidebar Reservations item
   - In `OwnerSidebar`, read count using existing `useReservationCounts(currentOrganization?.id ?? null)`.
   - Attach badge only to the `Reservations` nav row.
   - Use `SidebarMenuBadge` for native sidebar styling.
   - Show badge only when count > 0.

3. Fix stale API comment for maintainability
   - Update `getPendingCount` router comment so it no longer claims only `PAYMENT_MARKED_BY_USER`.

## Files To Modify
- `src/lib/modules/reservation/services/reservation-owner.service.ts`
  - `getPendingCount(...)`: statuses -> `CREATED`, `AWAITING_PAYMENT`, `PAYMENT_MARKED_BY_USER`.
- `src/features/owner/components/owner-sidebar.tsx`
  - Import and use `useReservationCounts`.
  - Render `SidebarMenuBadge` on the Reservations item when count is positive.
- `src/lib/modules/reservation/reservation-owner.router.ts`
  - Update `getPendingCount` procedure comment to match real behavior.

## Verification (end-to-end)
1. Static checks
   - Run `pnpm lint`.

2. Manual validation in app
   - Run `pnpm dev` and open owner dashboard.
   - Verify sidebar `Reservations` badge equals Reservations page `Inbox` tab count.
   - Confirm `AWAITING_PAYMENT` reservations are included in both counts.
   - Confirm `CONFIRMED`, `CANCELLED`, and `EXPIRED` do not affect badge.
   - Perform reservation actions (accept/reject/confirm payment) and confirm badge updates after mutation invalidation.
   - Switch organizations and confirm badge tracks selected organization.
   - Confirm no badge is rendered when count is `0`.
