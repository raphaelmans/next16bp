# Owner-only payment reminder in reservations

## Goal
- Ensure the "Payment methods missing" reminder is not shown in player view and appears only in owner view.

## Root cause
- `src/app/(auth)/reservations/page.tsx` (player reservations page) currently renders `OwnerPaymentMethodReminder`, which is owner-focused UI copy and CTA.

## Recommended implementation
1. Remove owner reminder from player reservations page.
   - Update `src/app/(auth)/reservations/page.tsx`:
     - remove `OwnerPaymentMethodReminder` import
     - remove `<OwnerPaymentMethodReminder />` from the page body

2. Render the same reminder in owner reservations page.
   - Update `src/app/(owner)/owner/reservations/page.tsx`:
     - add `OwnerPaymentMethodReminder` to owner component imports
     - render `<OwnerPaymentMethodReminder />` in the main content stack under the page header and above filters/tabs

3. Keep reminder logic unchanged.
   - Do not modify `src/features/owner/components/owner-payment-method-reminder.tsx`.
   - Existing internal checks (org presence/loading/error and no payment methods) are sufficient for this scope.

## Files to modify
- `src/app/(auth)/reservations/page.tsx`
- `src/app/(owner)/owner/reservations/page.tsx`

## Verification
- Run `pnpm lint`.
- Manual checks:
  - Player account at `/reservations`: confirm "Payment methods missing" is not shown.
  - Owner account at `/owner/reservations` with zero payment methods: confirm reminder appears with settings CTA.
  - Owner account at `/owner/reservations` with at least one payment method: confirm reminder is hidden.
  - Owner paid-offline dialog still blocks when no active payment methods (existing behavior unchanged).

## Note
- There is an existing distinction where this banner checks total methods while paid-offline flow checks active methods. Leave as-is for this targeted fix.
