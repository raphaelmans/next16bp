# Owned/Transferred Venue -> Curated Plan

## Goal
Add a first-class, safe transition for `RESERVABLE/CLAIMED` venues back to `CURATED/UNCLAIMED`, so this lifecycle is supported end-to-end:

`curated -> transferred/claimed -> curated`

## What Exists Today (Confirmed)
- Venue ownership state is stored on `place` (`organizationId`, `placeType`, `claimStatus`) in `src/lib/shared/infra/db/schema/place.ts`.
- Admin transfer only moves forward to owned (`RESERVABLE` + `CLAIMED`) in `src/lib/modules/court/services/admin-court.service.ts`.
- Claim approval currently always sets owned state, even for removal requests, in `src/lib/modules/claim-request/use-cases/approve-claim-request.use-case.ts`.
- There is no explicit admin mutation for owned -> curated in `src/lib/modules/court/admin/admin-court.router.ts`.

## Recommended Implementation

### 1) Add explicit admin recuration mutation
- Add a new DTO schema (include `placeId` and required `reason`) at `src/lib/modules/court/dtos/recurate-place.dto.ts` and export it from `src/lib/modules/court/dtos/index.ts`.
- Add `admin.court.recurate` mutation in `src/lib/modules/court/admin/admin-court.router.ts`.
- Implement `recuratePlace` in `src/lib/modules/court/services/admin-court.service.ts` with row lock (`findByIdForUpdate`) and transactional update:
  - `organizationId: null`
  - `placeType: "CURATED"`
  - `claimStatus: "UNCLAIMED"`
- In the same transaction, reset verification/bookability via `placeVerificationRepository.upsert(...)` in `src/lib/modules/place-verification/repositories/place-verification.repository.ts` to:
  - `status: "UNVERIFIED"`
  - `reservationsEnabled: false`
  - clear verified metadata (`verifiedAt`, `verifiedByUserId`, `reservationsEnabledAt`).
- Add/adjust domain errors in `src/lib/modules/court/errors/court.errors.ts` for invalid state (already curated, place not found as needed).
- Log business event `place.recurated` with `placeId`, `adminUserId`, `fromOrganizationId`, and `reason`.

### 2) Make removal approval actually re-curate
- Update `src/lib/modules/claim-request/use-cases/approve-claim-request.use-case.ts` to branch by `claimRequest.requestType`:
  - `CLAIM` -> keep existing behavior (`RESERVABLE` + `CLAIMED` + org assignment).
  - `REMOVAL` -> apply the same recuration transition (`CURATED` + `UNCLAIMED` + org null) and verification reset.
- Wire any needed dependency in `src/lib/modules/claim-request/factories/claim-request.factory.ts` (inject place verification repository/service dependency into the approve use case).
- Keep existing claim-request audit records (`claim_request` status and `claim_request_event`) unchanged, but update log message text so it reflects removal behavior correctly.

### 3) Expose recuration action in admin venue UI
- Add `useRecuratePlace` hook in `src/features/admin/hooks.ts` that calls `trpc.admin.court.recurate.useMutation` and invalidates:
  - `admin.court.list`
  - `admin.court.getById({ placeId })`
- Extend `src/features/admin/components/admin-court-ownership-transfer-card.tsx` with a "Return to Curated" action and confirmation dialog collecting `reason`.
- Wire action handler in `src/features/admin/components/admin-court-edit-view.tsx`.

### 4) Align claim review copy with backend behavior
- Update removal approval copy in `src/features/admin/components/claim-review-actions.tsx` so it states "returned to curated / ownership removed" (remove unsupported claims like automatic reservation cancellation unless implemented).
