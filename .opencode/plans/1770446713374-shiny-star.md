# Plan: Isolated early breakpoint for owner reservations tab content

## Goal
Prevent horizontal scrolling around ~1250px for the owner reservations tab panel inside `DashboardLayout` + `SidebarInset`, while keeping the change isolated to this feature component only.

## Recommended approach
Use Tailwind default breakpoint tokens and move this feature's table/card switch from `md` to `xl` so the card layout is shown below `1280px` (including ~1250px).

This keeps global behavior unchanged:
- No change to sidebar/mobile logic in `src/components/ui/sidebar.tsx`
- No change to layout breakpoints in `src/components/layout/dashboard-layout.tsx`
- No change to shared table primitive in `src/components/ui/table.tsx`

## Files to modify
1. `src/features/owner/components/reservations-table.tsx`

## Planned edits
1. In `src/features/owner/components/reservations-table.tsx`, update the responsive wrappers:
   - Mobile cards wrapper: `md:hidden` -> `xl:hidden`
   - Desktop table wrapper: `hidden md:block rounded-md border` -> `hidden xl:block rounded-md border`
2. Keep all other table/card internals unchanged.

## Why this solves it
- Around ~1250px, the content area is narrower due to sidebar + page padding.
- The desktop reservations table uses many `whitespace-nowrap` cells and naturally overflows.
- Showing the existing card layout until `xl` avoids that overflow at ~1250px.
- Scope is strictly local to the reservations component.
