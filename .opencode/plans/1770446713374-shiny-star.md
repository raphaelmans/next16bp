# Plan: Isolated early breakpoint for owner reservations tab content

## Goal
Prevent horizontal scrolling around ~1250px for the owner reservations tab panel inside `DashboardLayout` + `SidebarInset`, while keeping the change isolated to this feature component only.

## Recommended approach
Use Tailwind default breakpoint tokens and move this feature's table/card switch from `md` to `xl` so the card layout is shown below `1280px` (including ~1250px).

This keeps global behavior unchanged:
- No change to sidebar/mobile logic in `src/components/ui/sidebar.tsx`
- No change to layout breakpoints in `src/components/layout/dashboard-layout.tsx`
- No change to shared table primitive in `src/components/ui/table.tsx`

## Files to modify
1. `src/features/owner/components/reservations-table.tsx`

## Planned edits
1. In `src/features/owner/components/reservations-table.tsx`, update the responsive wrappers:
   - Mobile cards wrapper: `md:hidden` -> `xl:hidden`
   - Desktop table wrapper: `hidden md:block rounded-md border` -> `hidden xl:block rounded-md border`
2. Keep all other table/card internals unchanged.

## Why this solves it
- Around ~1250px, the content area is narrower due to sidebar + page padding.
- The desktop reservations table uses many `whitespace-nowrap` cells and naturally overflows.
- Showing the existing card layout until `xl` avoids that overflow at ~1250px.
- Scope is strictly local to the reservations component.

## Verification
1. Run app locally (`pnpm dev`) and open owner reservations page.
2. Confirm behavior by viewport width:
   - `~1250px` and `1279px`: card layout is rendered, no horizontal scroll in the reservations tab content.
   - `1280px+`: desktop table is rendered.
   - `<=768px`: mobile behavior still works as expected.
3. Functional smoke checks in both modes:
   - Tabs switch correctly.
   - Row expand/collapse and action buttons still work.
4. Run `pnpm lint`.

# Plan: Availability Studio 2xl-gated Week view + less cramped Day view

## Goal
1. Make Week view available only at Tailwind `2xl` (`1536px+`). Below `2xl`, force Day view.
2. Fix the Day view “cramped timeline” issue on `xl` and below by removing the right “Blocks” side card until `2xl`.
3. Keep the changes isolated to these pages/components (no global Tailwind breakpoint config changes).

## Recommended approach
- Add a page-local `is2xlUp` media-query boolean (`(min-width: 1536px)`) and derive `canUseWeek = is2xlUp`.
- Gate rendering so Week UI never mounts below `2xl` (even if `view=week` is in the URL).
- Hide the Day/Week ToggleGroup until `2xl`.
- In Day view, use a 2-column layout at `lg/xl` (left panel + timeline), and only add the right “Blocks” column at `2xl+`.
- Keep “Blocks” accessible below `2xl` by reusing the existing in-card blocks list (the `MobileDayBlocksList`) up to `2xl`.

## Files to modify
1. `src/app/(owner)/owner/bookings/page.tsx`
2. `src/app/(owner)/owner/places/[placeId]/courts/[courtId]/availability/page.tsx`

Notes:
- `src/app/(owner)/owner/venues/[placeId]/courts/[courtId]/availability/page.tsx` re-exports the `owner/places/...` file.

## Planned edits
### A) Add a local 2xl media-query gate
In each page file:
1. Add a small `useIs2xlUp()` hook (client-only `matchMedia("(min-width: 1536px)")`).
2. Compute `const canUseWeek = useIs2xlUp();`.
3. Update the view render condition:
   - from: `{isWeekView ? <Week /> : <Day />}`
   - to: `{canUseWeek && isWeekView ? <Week /> : <Day />}`
4. Add an effect so URL/view state is sanitized when shrinking below `2xl`:
   - if `!canUseWeek && view === "week"`, call `setViewParam("day")`.

### B) Make the Day/Week toggle only appear at 2xl
1. `src/app/(owner)/owner/bookings/page.tsx`
   - Change ToggleGroup: `className="hidden lg:flex"` -> `className="hidden 2xl:flex"`.
2. `src/app/(owner)/owner/places/[placeId]/courts/[courtId]/availability/page.tsx`
   - Change ToggleGroup: `className="hidden lg:flex"` -> `className="hidden 2xl:flex"`.

### C) Day view: remove the right Blocks side card until 2xl
For each page’s Day view section:
1. Change the Day grid columns:
   - from: `lg:grid-cols-[280px_minmax(0,1fr)_320px]`
   - to: `lg:grid-cols-[280px_minmax(0,1fr)] 2xl:grid-cols-[280px_minmax(0,1fr)_320px]`
2. Change the right Blocks card visibility:
   - from: `className="hidden lg:block"`
   - to: `className="hidden 2xl:block"`
3. Ensure Blocks are still reachable below `2xl`:
   - Change the existing blocks list wrapper:
     - from: `className="lg:hidden"`
     - to: `className="2xl:hidden"`

## Why this solves it
- Uses the default Tailwind breakpoint (`2xl = 1536px`) and keeps it local to these pages.
- Prevents Week view from rendering on widths where it’s cramped.
- Removes the right Blocks column on `xl` and below, giving the Day Timeline card the width it needs.

## Verification
1. `pnpm dev` and open:
   - `/owner/bookings`
   - `/owner/venues/[placeId]/courts/[courtId]/availability`
2. Width checks:
   - `1280px` and `1440px`: no Week toggle; Week view cannot render; Day Timeline is wider (no right Blocks column).
   - `1536px+`: Week toggle appears; Week view works; Day view shows right Blocks column.
3. Resize test:
   - Start at `2xl+` in Week view, shrink below `2xl`: view auto-switches to Day.
4. Smoke test:
   - Block selection + create flow still works.
   - Blocks list is accessible below `2xl`.
5. Run `pnpm lint`.
