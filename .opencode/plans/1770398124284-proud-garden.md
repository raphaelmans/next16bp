# Plan: Fix booking flash + booking ID overflow

## User goal
- Fix the transient `Booking details missing` flash after clicking Reserve, before the booking summary appears.
- Fix mobile overflow for Booking ID in reservation actions so long IDs truncate and do not cause horizontal scrolling.

## Scope and files to update
- `src/app/(auth)/places/[placeId]/book/page.tsx`
- `src/features/reservation/components/reservation-actions-card.tsx`
- `src/components/ui/card.tsx` will be reviewed for impact, but no global primitive change is planned unless required.

## Root causes
- Booking page currently gates skeleton on place/profile loading only. Availability can still be loading, so `selectedSlot` is temporarily `undefined` and triggers `Booking details missing`.
- Booking ID row uses flex + long unbreakable token; `truncate` is present but missing `min-w-0` constraints in the flex chain, so the text does not shrink correctly on small screens.

## Implementation plan
1. In `src/app/(auth)/places/[placeId]/book/page.tsx`, add explicit booking-resolution guards:
   - Compute a `hasResolvableInputs` boolean from parsed query values (`startTime`, mode-dependent `sportId`/`courtId`, valid duration/date context).
   - Compute `isResolvingAvailability` from `availabilityQuery` flags (`isLoading`/`isFetching` while inputs are resolvable).
2. Adjust render order so transient empty data does not show as an error:
   - Keep existing top-level skeleton for `isLoading || isLoadingProfile`.
   - Add an availability-stage skeleton condition before the missing-details branch: when inputs are resolvable and availability is still resolving, render `PlaceBookingSkeleton`.
   - Show `Booking details missing` only after availability resolution is complete and `selectedSlot` is still missing.
3. Preserve existing non-bookable behavior:
   - Keep `!place || !showBooking` branch unchanged so truly unavailable venues still show `Bookings not available`.
4. In `src/features/reservation/components/reservation-actions-card.tsx`, make Booking ID row shrink-safe on mobile:
   - Add `min-w-0` to the Booking ID row container (`flex ...`).
   - Add `min-w-0` (and explicit `overflow-hidden` if needed) to the `<code>` element while keeping `flex-1 truncate`.
   - Keep copy button `shrink-0` so action remains visible/tappable.
   - If needed, add `min-w-0` on local `<Card>` / `<CardContent>` className usage in this component (not global primitive changes).
5. Do not change `src/components/ui/card.tsx` globally unless local class fixes prove insufficient, to avoid side effects across all cards.

## Verification
- Repro flow: open a venue, select slot, click Reserve, and confirm booking page does not flash `Booking details missing` before summary appears.
- Negative path: open booking URL with invalid/missing slot params; confirm `Booking details missing` still appears after loading settles.
- Slow network check: throttle and repeat Reserve flow; confirm stable skeleton then summary.
- Mobile check (320-375px): on reservation detail page, ensure Booking ID truncates, copy button remains visible, and no horizontal page scroll appears.
- Run `pnpm lint`.

## Risk notes
- Overly broad availability-loading gating could keep skeleton visible too long; guard with resolvable-input checks to avoid indefinite loading.
- Keeping the missing-details branch after query settlement preserves fallback UX for stale/invalid deep links.
