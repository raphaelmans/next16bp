# Plan: Add owner onboarding dashboard action

## Goal
Add a "Go to dashboard" action from the owner onboarding header that sends users to the owner dashboard route (`/owner`, implemented by `src/app/(owner)/owner/page.tsx`).

## Recommended implementation
1. Update `src/components/layout/onboarding-shell.tsx`.
   - Extend `OnboardingShell` props to accept an optional `dashboardHref?: string`.
   - In the existing header action row, render a new `Button` (`variant="ghost"`, `size="sm"`, `asChild`) with a `Link` labeled "Go to dashboard" when `dashboardHref` is provided.
   - Keep current `Player View` and `Sign Out` actions unchanged.

2. Update `src/app/(owner-onboarding)/layout.tsx`.
   - Pass `dashboardHref={appRoutes.owner.base}` into `OnboardingShell`.
   - Keep session enforcement and metadata logic unchanged.

## Rationale
- This satisfies the request at the layout level while keeping `OnboardingShell` reusable.
- Using `appRoutes.owner.base` avoids legacy `/dashboard` behavior and guarantees routing to owner dashboard.
- Styling stays consistent with existing onboarding header actions (aligned with the current design system and `frontend-design` intent for polished UI consistency).

## Verification
1. Run `pnpm lint` from repo root.
2. Run `pnpm dev` and open an owner onboarding route (for example `/owner/get-started`).
3. Confirm the header shows `Go to dashboard`.
4. Click it and verify navigation lands on `/owner` (`src/app/(owner)/owner/page.tsx`).
5. Confirm `Player View` and `Sign Out` still work and header layout remains usable on mobile widths.
