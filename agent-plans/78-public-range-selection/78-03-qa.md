# Phase 3: QA + Polish

Status: draft

## Objective

Verify selection correctness (range math, blocked behavior, URL state), accessibility (reduced motion, keyboard), and booking flow integrity.

## Automated

- [ ] `pnpm lint`
- [ ] `TZ=UTC pnpm build`

## Manual Test Matrix

Range selection

- [ ] Drag-select 2h/3h range and confirm CTA uses correct `duration`.
- [ ] Drag “upwards” (end before start) and ensure start/end normalize correctly.
- [ ] Attempt to drag across a blocked slot (maintenance/booked/walk-in) and confirm behavior matches spec (prevent/clamp).
- [ ] Confirm max duration (24h) enforced.

URL state

- [ ] Refresh preserves selection (`startTime`, `duration`).
- [ ] Switching day clears selection.

Time zone

- [ ] Confirm `dayKey` and startTime stay aligned to place time zone.

Motion + accessibility

- [ ] `prefers-reduced-motion` yields minimal/no transform animation.
- [ ] Keyboard fallback works to form a range.

Booking flow

- [ ] Continue navigates to booking confirm page with correct params.
- [ ] Guest login return-to preserves selection.
