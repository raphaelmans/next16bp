## Manual Test Matrix

Routing + canonical

- [ ] `/venues/<uuid>/courts/<courtId>` redirects server-side to `/venues/<slug>/courts/<courtId>`.
- [ ] Redirect preserves query params (eg `?view=day&date=2026-01-29`).
- [ ] `/venues/<slug>/courts/<courtId>` returns 404 when court does not belong to venue.

Availability

- [ ] Default load uses `view=month` and a clamped `month`/`date` within booking window.
- [ ] Switching Month ↔ Day keeps selection coherent and clears `startTime` when appropriate.
- [ ] Duration changes clear `startTime` and refresh availability.
- [ ] Booking window exceeded error surfaces the correct “jump to latest available date” UX.

Booking flow

- [ ] Selecting a slot enables CTA.
- [ ] Authenticated: CTA navigates to `/venues/<slug>/book?...`.
- [ ] Guest: CTA navigates to `/login?redirect=<returnTo>` and returns to the same court page + selection.

Time zone

- [ ] With `place.timeZone != UTC`, day boundaries are correct (no off-by-one day in availability).

Motion + a11y

- [ ] `prefers-reduced-motion` disables transform-heavy animations (instant or opacity-only).
- [ ] Keyboard navigation still works for toggles/buttons.

Mobile

- [ ] Sticky bottom CTA behaves correctly when a slot is selected.
