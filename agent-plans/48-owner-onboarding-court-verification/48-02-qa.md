# Phase 2: QA + Regression

**Dependencies:** Phase 1 complete  
**Parallelizable:** No  
**User Stories:** US-02-08

---

## Objective

Validate the new onboarding redirect flow and ensure no regressions to existing court management routes.

---

## Regression Areas

- Existing court setup wizard still works:
  - `/owner/places/{placeId}/courts/setup`
  - `/owner/places/{placeId}/courts/{courtId}/schedule`
  - `/owner/places/{placeId}/courts/{courtId}/slots`
- Existing "Add Court" entry points still route to the wizard (not the one-time page).
- Verification pages remain reachable and functional:
  - `/owner/verify`
  - `/owner/verify/{placeId}`

---

## Manual QA Script

1. Create a new place:
   - Confirm redirect to `/owner/places/{placeId}/courts/new`.
2. Create a court:
   - Confirm success toast.
   - Confirm redirect to `/owner/verify/{placeId}`.
3. Navigate back to courts list:
   - Confirm the court appears.
4. From courts list click "Add Court":
   - Confirm it still uses the wizard route.

---

## Build Validation

- `pnpm lint`
- `pnpm build`
- Optional timezone regression check: `TZ=UTC pnpm build`
