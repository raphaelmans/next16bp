# Phase 5A - QA

## Goal

Validate multi-file import end-to-end (upload -> normalize -> review -> commit) and ensure migrations + lint/build are clean.

---

## Shared / Contract

- [ ] Confirm error messages are user-safe and actionable for:
- [ ] too many files
- [ ] invalid types
- [ ] AI already used
- [ ] AI required (images)

---

## Server / Backend

Validation checklist:
- [ ] Migration applies and backfills sources for existing jobs.
- [ ] `discardJob` deletes all source files (best-effort).
- [ ] `normalize` merges sources and produces stable row counts.
- [ ] AI mode:
- [ ] denies when venue AI already used
- [ ] consumes AI only on successful normalization

---

## Client / Frontend

Manual flows:
- [ ] Upload 3 images -> normalize (AI) -> review -> fix -> commit
- [ ] Upload CSV+XLSX+ICS -> normalize deterministic -> review -> commit
- [ ] Upload CSV only -> normalize AI -> ensure mapping improves over heuristic
- [ ] Venue with AI already used:
- [ ] upload still allowed
- [ ] AI normalize disabled
- [ ] deterministic normalize still works (no images)
