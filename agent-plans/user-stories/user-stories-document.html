Key Decisions

  • Model split: Place = venue/listing, Court = bookable unit (1 court = 1 sport).
  • Granularity: Booking durations are multiples of 60; slots are published in 60-minute increments.
  • Any-available selection: choose the lowest total price option across matching courts.
  • Overnight support: court hours can span midnight and remain bookable.

Domain: 14-place-court-migration

Overview

This domain covers the product and UX changes required to move from the legacy model where a single “court” represented a location, into the v1.2 model:

  • Place = venue/location listing (multi-sport capable)
  • Court = a single bookable unit inside a place (1 court = 1 sport)
  • Slot = availability for a specific court

It includes player-facing UX (choose sport → choose specific court or “Any available” → book), owner tooling (manage places, courts, hours, pricing rules, slots), and migration continuity.
