# Stepper v2: QA Checklist

## Happy Path

1) Visit `/owners/get-started` logged out
2) CTA -> signup -> land on `/owner/get-started`
3) Create organization -> hub updates (Step 1 complete)
4) Add venue -> submit -> returns to `/owner/get-started` (Step 2 complete)
5) Click “Get your venue verified” -> submit docs -> returns to hub (Step 3 pending)
6) Click “Set up courts” -> create first court -> hub reflects completion rule
7) Import bookings flow reachable from hub (Step 4B)

## Edge Cases

- Claim listing submitted (pending) shows Step 2 as complete, but Step 3/4 remain disabled until a venue exists.
- Multiple venues: hub selects a consistent “current venue” for steps 3/4 (document selection logic).
- Refresh behavior: hub reflects updated statuses without requiring hard reload.
