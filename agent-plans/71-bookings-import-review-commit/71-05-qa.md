## QA Checklist

- Upload creates a resumable job and redirects to `/owner/import/bookings/[jobId]`
- AI usage status is visible per venue and consistent across users
- AI normalize requires explicit one-time confirmation
- Draft rows render with clear per-row error states
- Owner can edit court/start/end/reason and errors recompute correctly
- Owner can delete rows
- Commit is blocked when any error rows remain
- Commit creates maintenance blocks and prevents booking overlaps
- Commit result summary is accurate (created/skipped/failed)
- Retry commit is idempotent (already committed rows are skipped)
- Mobile usability: table scroll, dialogs usable
- Accessibility: keyboard nav, focus management, readable errors
