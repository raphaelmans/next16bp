# Phase 3: QA

**Dependencies:** Phase 1 + Phase 2

---

## Objective

Validate the 60-day window is consistent across public availability, reservation creation, and owner tooling.

---

## Manual QA

- Public place booking
  - [ ] Date picker cannot select a date > 60 days ahead
  - [ ] If a deep link sets a date > 60 days ahead, UI shows empty/error state and server rejects fetch

- Public schedule page
  - [ ] Month view cannot navigate beyond 60-day window
  - [ ] Range endpoints reject if the computed month end exceeds the cap

- Booking creation
  - [ ] `reservation.createForCourt` rejects > 60 days
  - [ ] `reservation.createForAnyCourt` rejects > 60 days
  - [ ] `reservation.create` (by slotId) rejects if slot start is > 60 days

- Owner slots
  - [ ] Bulk slot modal cannot select dates beyond 60 days
  - [ ] Server rejects any create/createBulk requests beyond 60 days
