Key Design Patterns

Outbox Pattern
Jobs written in the same DB transaction as the business entity — guarantees delivery if commit succeeds.
Idempotency
Unique key per event+recipient+channel. Database constraint prevents duplicate jobs; Resend header prevents duplicate emails.
Batch Claiming
Cron claims 25 jobs per run with SELECT ... FOR UPDATE to prevent double-processing across concurrent runs.
Graceful Degradation
Disabled channels are logged and skipped. Invalid subscriptions/tokens auto-revoked and marked SKIPPED.
