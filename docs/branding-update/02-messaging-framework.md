# Messaging Framework

## One-liner

**Court discovery for players. Free reservation system for venues.**

## Short pitch (2–3 sentences)

KudosCourts helps players discover sports courts and reserve time with less back-and-forth. For venue owners, it’s a **free reservation system** designed for simple setup, verified trust, and fewer booking conflicts—without payment processing or long contracts.

## Tagline

Keep: **Discover. Reserve. Play.**
Add as descriptor where space allows: **Free for venues.**

## Key messages (Players)

- Stop chasing schedules in DMs—see availability and reserve faster.
- Find courts by sport and location across the Philippines.

## Key messages (Venue owners)

- Free to list and accept online reservations (core).
- No payment processing: keep existing payment methods.
- Reduce double-bookings with booking imports + a single dashboard.
- Verification builds trust; owners control when reservations go live.

## Objections + Responses

- “How is it free?” → Core is free; we aim to sustain via ads/sponsored placements.
- “Do you take payments?” → No processing; players pay venues directly.
- “Is setup hard?” → Guided onboarding + import bookings to start clean.
- “Will I lose control?” → Owners choose when to enable reservations and can manage availability.

## Terminology

- Use **rates** (venue charges) / **price** (total for a slot) on public pages.
- Avoid public “Pricing” (SaaS plan implication).
- Keep “pricing rules” terminology inside owner ops only (optional future rename to “rate rules”).
